# 前端与后端对接说明

## 1. 当前状态

- **登录 / 注册**：已对接后端。
  - 登录：`POST /api/auth/login`（学号或邮箱 + 密码），返回 `token`、`data`（用户信息），前端存于 `localStorage`。
  - 注册：`POST /api/auth/register`（学生：邮箱 @xmu.edu.my、用户名、密码），成功后同样存 token 与用户信息。
- **帖子、评论**：目前为**前端 mock 数据**（`mockPosts.js`、`mockComments.js`），**未持久化**。
  - 因此：评论、发帖后刷新或重新打开，数据会恢复为 mock，不会保留。

## 2. 如何让评论/帖子持久化（连通后端数据库）

登录注册完善后，只要后端提供以下接口，前端即可替换 mock、实现持久化：

1. **帖子**
   - 列表：`GET /api/posts`（或分页参数）
   - 单条：`GET /api/posts/:id`
   - 发布：`POST /api/posts`（需带 token，body：content、图片等）
   - 可选：删除 `DELETE /api/posts/:id`

2. **评论**
   - 某帖评论列表：`GET /api/posts/:id/comments`
   - 发表评论：`POST /api/posts/:id/comments`（需带 token，body：content、replyTo 等）

3. **点赞**
   - 帖子点赞：`POST /api/posts/:id/like` 或 `PUT /api/posts/:id/like`（需带 token）

前端需要做的改动：

- 在 `src/api/` 下增加 `posts.js`、`comments.js` 等，用 `fetch` 调用上述接口，并带上 `Authorization: Bearer <token>`。
- 将 `TreeHole`、`PostDetail`、`PostNew`、`MyPosts` 等页面中使用的 `MOCK_POSTS`、`getCommentsForPost` 等替换为上述 API 调用。

## 3. 小结

| 功能       | 是否已连后端 | 说明 |
|------------|--------------|------|
| 登录 / 注册 | 是           | 已调用现有 auth 接口 |
| 帖子列表/详情 | 否           | 使用 mock，需后端提供帖子接口后替换 |
| 评论 / 点赞 | 否           | 使用 mock，需后端提供评论/点赞接口后替换 |
| 我的帖子   | 否           | 基于 mock 的 authorId 筛选，后端可提供「当前用户帖子」接口 |

完成上述接口后，前端即可完全连通后端数据库，评论与帖子会持久化保存。
