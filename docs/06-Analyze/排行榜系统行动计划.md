# 排行榜系统行动计划

> 基于食堂商品点评的五大榜单与评分规则，含技术实现步骤与数据表设计。

---

## 一、点评等级与分值

| 等级     | 分值 w_i |
|----------|----------|
| 夯爆了   | 10       |
| 顶级     | 7        |
| 人上人   | 4        |
| NPC      | 1        |
| 拉完了   | -1       |

**约定**：仅统计**一级点评**（`product_comments.parent_id IS NULL`）参与评分与榜单；回复（二级）不计入。

---

## 二、菜品综合评分计算

**公式**：

$$S_{dish} = \frac{\sum_i (n_i \times w_i)}{\sum_i n_i}$$

- **S_dish**：该商品（菜品）的综合评分  
- **n_i**：该商品各等级点评数（夯爆了/顶级/人上人/NPC/拉完了 的条数）  
- **w_i**：对应等级分值（10 / 7 / 4 / 1 / -1）  
- **综合评分 = 总得分 / 总点评数**，保留 2 位小数。
- **边界**：若该商品总点评数为 0（分母为 0），不参与按综合评分排序的榜单；S_dish 可为 NULL 或不计入排名。

---

## 三、商家综合评分计算

**关键要求**：不能直接用累计得分，否则点评数多的商家容易偏高。

**推荐算法**：按菜品加权平均。

**公式**：

$$S_{shop} = \frac{\sum_{dish} (S_{dish} \times N_{dish})}{\sum_{dish} N_{dish}}$$

- **S_shop**：该商家的综合评分  
- **S_dish**：该商家下某道菜的综合评分  
- **N_dish**：该菜的点评数  
- **优点**：既考虑菜品评分，也平衡点评数量差异。  
- 保留 2 位小数。
- **边界**：若店铺下无商品或全部商品点评数为 0（分母为 0），该店铺不参与最夯商家榜；S_shop 可为 NULL。

---

## 四、每次点评后需自动维护的数据

用户对**商品**完成一次一级点评（新增或逻辑删除需回滚统计）后，系统需：

| 序号 | 维护项 | 说明 |
|------|--------|------|
| 1 | 该商品各等级点评数累计 | 5 个等级的条数 |
| 2 | 该商品总点评数 | 一级点评总数 |
| 3 | 该商品综合评分 S_dish | 按第二节公式计算 |
| 4 | 该店铺各等级点评数累计 | 店铺下所有商品的一级点评按等级汇总 |
| 5 | 该店铺总点评数 | 店铺下所有一级点评总数 |
| 6 | 该商家综合评分 S_shop | 按第三节公式计算（**不按周重置**，见下） |
| 7 | **该店铺当周点评数** | 用于门庭若市榜；每周一 0 点（东八区）清零，点评时 ±1 |
| 8 | **该用户当周点评数** | 用于点评达人榜与个人空间展示；每周一 0 点（东八区）清零，点评时 +1 |

**说明**：最夯单品榜使用**上线至今的累计** S_dish（上述 1～3），无需单独维护「本周」商品评分。  

**逻辑删除点评时**：上述 1～8 需反向扣减/重算（当周数也要扣减），保证数据一致。

## 五、榜单定义与周期规则（★ 必读）

### 5.1 榜单一览

| 榜单 | 规则 | 数量 |
|------|------|------|
| **1. 最夯单品榜** | 评价标准为该商品**上线至今的累计综合评分**（即全量 S_dish）；同分按上架时间越早越前；榜单每周一更新（东八区 0 点） | Top 5 |
| **2. 门庭若市商家榜** | 按**当周点评量**从多到少（每周一结） | Top 5 |
| **3. 最夯商家榜** | 按商家综合评分从高到低；**评分不重置**，榜单每周一更新 | Top 5 |
| **4. 爆款新品** | **商家上架时间**（商品创建时间）&lt; 7 天且综合评分高；**每周一重置** | Top 3 |
| **5. 点评达人榜** | 按**每周点评数**从多到少（用户当周点评数）；**每周一重置** | Top 5 |

### 5.2 ★ 周期与重置规则（强调）

**时间基准**：所有「每周一 0 点」均以**东八区（Asia/Shanghai）**时间为准。

- **最夯单品榜**  
  - 评价标准 = 该商品**上线至今的累计综合评分**（全量 S_dish，即第二节公式用该商品全部历史一级点评各等级数计算），不按周重置。  
  - 榜单每周一 0 点（东八区）刷新排名结果。

- **点评达人榜**  
  - 统计的是**用户当周的点评数**。  
  - **每次用户发表一级点评时，必须更新该用户的「当周点评数」**（+1；逻辑删除时 -1）。  
  - **每周一 0 点（东八区）**对该计数做**重置**（清零），新的一周重新累计。  
  - 榜单 = 当周点评数 Top 5。  
  - **用户当周点评数**可在**个人空间**展示，由前端读取接口展示（后端只需提供该字段或接口）。

- **爆款新品**  
  - **上架时间** = 商家上架该商品的时间（即商品表 `created_at`）。  
  - **新品定义**：**created_at 距今 ≤ 7 个自然日**的商品（**按东八区时间**计算「距今」）；每周一刷新榜单时按此条件筛选后按**累计综合评分** S_dish 取 Top 3。  
  - **每周一重置**：指榜单结果重算，非重置商品属性；名额不足可为空（0～3 条）。

- **最夯商家榜**  
  - **商家综合评分 S_shop 不重置**：随历史点评经年累月变化，不做每周清零。  
  - **榜单**：**每周一更新**一次排名（按当前 S_shop 取 Top 5），即「每周一榜单重置/刷新」，但用的仍是长期累积的 S_shop。

- **门庭若市商家榜**  
  - **每周一结**：只统计**当周**该店铺收到的点评数量（即「店铺当周点评数」）。  
  - 每次店铺下有新一级点评时，该店铺的**当周点评数** +1；逻辑删除时 -1。  
  - **每周一 0 点（东八区）**对店铺当周点评数做**重置**，新的一周重新累计。  
  - 榜单 = 当周点评数 Top 5。

### 5.3 同分时的排序与名额不足

- **同分（保留 2 位小数后仍相同）时**：按**上架/上线时间**排序，**上线时间越久的排在前面**（即 `created_at` 越早的排越前）。适用于按评分或综合评分的榜单（如最夯单品、最夯商家、爆款新品等）；若为“按数量”的榜（门庭若市、点评达人），同数量时也可按首次达标时间或用户 id 等做次要排序，建议统一约定“时间早的在前”。
- **名额不足时**：若符合条件的条目不足 Top N（例如爆款新品没有上架 &lt; 7 天且评分高的商品），**榜单可为空或不足 N 条**，无需凑满，返回实际数量即可（如爆款新品返回 0、1 或 2 条）。

**通用规则**：

- 商品/店铺/用户若被**逻辑删除**，不参与、不展示在榜单中。  
- 综合评分、统计数均保留 2 位小数（或整数时取整）。  
- 设计上预留扩展：可后续新增榜单或统计维度。

## 六、技术实现思路

### 6.1 点评新增/修改时的更新顺序

1. **更新商品维度**  
   - 该商品各等级点评数 ±1（或按实际变更）  
   - 该商品总点评数  
   - 该商品综合评分 S_dish（按公式重算）

2. **更新店铺维度**  
   - 该店铺各等级点评数（汇总其下所有商品）  
   - 该店铺总点评数  
   - 该商家综合评分 S_shop（按公式重算，仅用该店铺下商品）  
   - **该店铺当周点评数**（+1 或逻辑删除时 -1）  
   - **该用户当周点评数**（+1 或逻辑删除时 -1）

建议：在**发表一级点评**和**逻辑删除一级点评**的两个接口中，统一调用「统计更新函数」，保证 1、2 顺序执行且事务内完成。

### 6.2 排行榜更新策略与「每周一」约定

- **每周一 0 点（或首班定时任务）**建议统一处理：  
  - **重置**：用户当周点评数、店铺当周点评数（清零，新的一周重新累计）。  
  - **榜单刷新**：最夯单品榜、门庭若市商家榜、最夯商家榜、爆款新品、点评达人榜的**排名结果**重新计算并写入排行榜表/缓存。

- **最夯商家榜**：商家综合评分 **不重置**，只每周一按当前 S_shop 重算 Top 5 并更新榜单快照。

- **门庭若市商家榜**：用**当周点评量**（店铺当周点评数）排序，数据来源为「每次点评时维护的店铺当周点评数」，每周一清零后重新累计。

- **爆款新品**：上架时间以**商品上架时间**（如 `products.created_at`）为准，每周一刷新榜单时筛选上架 &lt; 7 天且综合评分高的 Top 3。

- **点评达人榜**：用**用户当周点评数**排序，每次点评时更新该用户当周点评数，每周一清零；该数也可供**个人空间**展示（前端调用接口显示）。

### 6.3 注意事项（实现时必做）

- 商品综合评分、商家综合评分均**保留 2 位小数**（如 `DECIMAL(5,2)` 或输出前 toFixed(2)）。  
- 商品/店铺已逻辑删除（`deleted_at IS NOT NULL`）的，**不参与**任何榜单与统计。  
- **同分时**：按**上架/上线时间**排序，**越早上线越靠前**（`created_at` 升序作为第二排序键）。  
- **名额不足**：榜单可返回少于 N 条或空数组，不凑满（如爆款新品可为 0～3 条）。  
- **每周一 0 点**：统一以**东八区（Asia/Shanghai）**为准，用于自然周起止、当周数重置、榜单刷新及爆款新品「7 天」计算。  
- 系统设计需**兼容未来新增榜单或统计维度**（如独立统计表、配置化榜单类型、时间窗口参数等）。

## 七、数据库与接口规划（建议）

### 7.1 统计字段落表（建议新增/扩展）

- **products 表**（或单独 `product_stats`）：  
  - 各等级点评数：`count_rating_1`～`count_rating_5`（对应 夯爆了/顶级/人上人/NPC/拉完了）  
  - `review_count`：一级点评总数  
  - `comprehensive_score`：S_dish，DECIMAL(5,2)  
  - 上架时间用现有 `created_at` 即可（即商家上架时间）。

- **shops 表**（或单独 `shop_stats`）：  
  - 各等级点评数、`review_count`、`comprehensive_score`（S_shop），类型同上。  
  - **`weekly_review_count`**：当周点评数，**每周一 0 点重置为 0**，用于门庭若市榜。

- **用户维度**（如 `users` 表扩展或 `user_stats` 表）：  
  - **`weekly_comment_count`**：用户当周点评数，**每周一 0 点重置为 0**，用于点评达人榜与个人空间展示。

- **排行榜结果表**（可选，便于定时任务写入）：  
  - 如 `ranking_snapshots`：榜单类型、统计周期（如 week_start，每周一）、排名、关联 id（product_id/shop_id/user_id）、得分/数量、快照时间等。

### 7.2 接口建议

- **点评发表/删除**：在现有 `POST /api/canteen/products/:productId/comments` 与 `DELETE .../comments/:commentId` 内，在事务中更新：商品与店铺统计、S_dish、S_shop、**店铺当周点评数**、**用户当周点评数**。  
- **个人空间/用户信息**：提供用户当周点评数（如 `GET /api/users/me` 或 profile 接口中带上 `weekly_comment_count`），供前端在个人空间展示。  
- **排行榜查询**：  
  - `GET /api/canteen/rankings/hot-products`（最夯单品榜）  
  - `GET /api/canteen/rankings/busy-shops`（门庭若市商家榜）  
  - `GET /api/canteen/rankings/top-shops`（最夯商家榜）  
  - `GET /api/canteen/rankings/new-hit-products`（爆款新品）  
  - `GET /api/canteen/rankings/active-users`（点评达人榜）  
  返回前 N 名及得分/数量，排除已逻辑删除主体。

## 八、实施步骤（行动清单）

| 步骤 | 内容 | 优先级 |
|------|------|--------|
| 1 | 设计并执行迁移：products/shops 统计字段 + 店铺当周点评数、用户当周点评数 | 高 |
| 2 | 实现「点评后更新」逻辑：商品/店铺各等级数、总点评数、S_dish、S_shop、**店铺当周点评数、用户当周点评数** | 高 |
| 3 | 点评逻辑删除时回滚/重算上述统计（含当周数） | 高 |
| 4 | 定时任务：每周一 0 点重置「用户当周点评数」「店铺当周点评数」，并刷新各榜单快照 | 高 |
| 5 | 实现五大榜单（最夯单品/门庭若市/最夯商家/爆款新品/点评达人），最夯商家用长期 S_shop，门庭若市/达人用当周数 | 高 |
| 6 | 提供排行榜查询 API + 个人空间/用户接口中返回当周点评数，前端对接 | 中 |
| 7 | 预留扩展：新榜单类型、新统计维度可配置或加表 | 低 |
