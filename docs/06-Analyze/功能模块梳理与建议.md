## 一、模块全景

XMUMDorm 共含 **20 个功能模块**，按职责域分为 5 层：

```
┌──────────────────────────────────────────────────────┐
│                  用户入口层                            │
│  认证 (Auth)   │  我的/空间 (MyZone)  │  关于 (About) │
├──────────────────────────────────────────────────────┤
│                  社交内容层                            │
│  树洞 (Treehole)  │  广场 (Square)  │  社团 (Clubs)   │
├──────────────────────────────────────────────────────┤
│                  生活服务层                            │
│  食堂 (Canteen)   │  二手 (Marketplace)  │  跑腿 (Errands) │
├──────────────────────────────────────────────────────┤
│                  学习工具层                            │
│  一站通 (Handbook) │ 课程评价 (Course) │ 课表 (Schedule) │ 日记 (Diary) │ 待办 (Todos) │
├──────────────────────────────────────────────────────┤
│                  基础设施层                            │
│  通知 (Notify) │ 推送 (Push) │ 等级 (Level) │ 组织 (Org) │ 后台 (Admin) │ 举报 (Report) │
└──────────────────────────────────────────────────────┘
```

## 六、移动端 App 迁移开发顺序

以下按**严格顺序**排列每个模块的开发步骤。顺序依据：**用户核心行为路径 → 依赖关系 → 使用频率 → 开发成本**。

---

### 第一阶段：骨架与核心闭环（第 1 周，约 18h）

> **目标：** 用户能注册、登录、浏览树洞、发帖互动、查看个人主页——完整的"刷帖"体验闭环。

```
步骤 0 ─ 架构补完（2h）
  ├── app.json 注册 expo-linear-gradient / expo-blur 插件
  ├── App.tsx 包裹 QueryClientProvider
  └── 抽 TabNavigator 组件，解耦 App.tsx

步骤 1 ─ 认证系统（4h）★ 先决条件：步骤 0
  ├── 注册页 RegisterScreen（复用 LoginScreen 样式）
  ├── 游客模式（skipLogin → AsyncStorage）
  └── 登录态持久化增强（token 过期处理、自动刷新 /me）

步骤 2 ─ 树洞系统（8h）★ 先决条件：步骤 1
  ├── Tag 工具栏对接真实 API（替换假数据 DUMMY_TAGS）
  ├── Tag 管理面板 TreeHoleTagPanel（弹窗选择可见标签）
  ├── 帖子搜索 SearchScreen（/posts/search?q=）
  ├── 瀑布流性能优化（图片缩略图、虚拟列表）
  └── 下拉刷新 + 无限滚动完善

步骤 3 ─ 我的 / 个人空间（4h）★ 先决条件：步骤 1
  ├── MyZoneScreen 完整版：头像/昵称/等级/统计(帖子/点评/收藏)
  ├── ProfileEditScreen：修改昵称 + 头像上传
  ├── 退出登录 + 清理缓存
  └── 等级徽章 UserLevelBadge 组件
```

**第一阶段交付物：**
- ✅ 登录 → 树洞（Tag筛选+搜索+瀑布流）→ 帖子详情（点赞+评论）→ 发帖
- ✅ 我的主页 → 编辑资料 → 退出登录
- ✅ 注册 → 自动登录
- ✅ 骨架屏 + 空状态 + 错误处理
