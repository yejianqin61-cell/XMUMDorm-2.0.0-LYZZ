# 安全策略

## 认证与鉴权

| 策略 | 实现 | 说明 |
|------|------|------|
| 密码哈希 | `bcrypt` (salt rounds ≥ 10) | 绝不存储明文密码 |
| JWT 签名 | `JWT_SECRET` 环境变量，≥ 32 字符 | 私钥不可硬编码，不可提交 Git |
| Token 过期 | 默认 7 天 | 敏感操作（修改密码）需重新验证 |
| 角色体系 | `student \| merchant \| admin` | admin 即官方号，不可自行注册 |
| 路由守卫 | `middleware/auth.js` `authenticateToken` | 所有需登录接口必须挂载此中间件 |
| 管理员鉴权 | `middleware/adminAuth.js` `requireAdmin` | 管理员接口双重鉴权 |

## 网络安全

| 策略 | 实现 | 说明 |
|------|------|------|
| 请求限流 | `express-rate-limit` | API 全局 100 req/15min，登录/注册 5 req/15min |
| HTTP 安全头 | `helmet` | 所有响应自动添加安全头 |
| CORS | 白名单模式 | 仅允许前端域名 + 移动端 |
| SQL 注入防护 | 参数化查询 (`database.js` `query()`) | **禁止**字符串拼接 SQL |

## 输入安全

| 策略 | 实现 | 说明 |
|------|------|------|
| XSS 防护 | `sanitize-html` | 所有用户输入在入库前清洗 |
| 敏感词过滤 | `middleware/sensitiveWordFilter.js` | 发帖/评论/点评自动过滤 |
| 文件上传校验 | Multer + 白名单扩展名 | 仅允许 jpg / png / webp / gif，≤ 5MB |
| 请求体校验 | 路由层显式校验 | 必填字段缺失 → 400 |

## 审计与追溯

| 策略 | 实现 | 说明 |
|------|------|------|
| 操作审计 | `services/auditLog.js` `logAudit()` | 管理员所有操作必须记录 |
| 举报系统 | `routes/reports.js` | 用户可举报违规内容 |
| 制裁系统 | `middleware/checkSanction.js` | 封禁/禁言，到期自动解除 |
| 逻辑删除 | `deleted_at` 字段 | 数据不物理删除，可追溯恢复 |

## 禁止事项

1. ❌ 禁止在代码中硬编码密钥/密码/Token
2. ❌ 禁止使用字符串拼接构建 SQL
3. ❌ 禁止跳过鉴权中间件（除非明确标记为公开接口）
4. ❌ 禁止上传目录可执行脚本
5. ❌ 禁止在日志中打印密码/Token
6. ❌ 禁止将 `.env` 提交到 Git
