# 移动端约束

## 核心原则

> **移动端 1:1 复刻 Web 端 UI。后端 API 100% 复用，不因移动端分叉。**

---

## 技术栈硬约束

| 层面 | 技术 | 不可变原因 |
|------|------|-----------|
| 框架 | **Expo SDK 52+** | OTA 热更新、无需原生编译环境、团队已采纳 |
| 路由 | **Expo Router**（文件系统路由） | 与 Web React Router 结构对标 |
| 样式 | **StyleSheet.create** + **expo-blur** | 液态玻璃效果必须用 expo-blur 实现 |
| HTTP | **fetch**（同 Web） | API 层代码直接复用 |
| 状态管理 | **TanStack Query**（同 Web） | 与 Web 统一 |
| 图标 | **@expo/vector-icons** | Expo 内置 |

## 禁止使用的原生模块

以下模块除非经过审批，否则禁止引入：

- ❌ 原生导航库（必须用 Expo Router）
- ❌ 自定义原生桥接（除非 Expo 生态不支持且无可替代方案）
- ❌ 第三方推送 SDK（必须用 Expo Notifications）
- ❌ 原生数据库（SQLite 等 —— 数据必须走后端 API）

## 代码复用策略

```
Web (frontend/src/)          Mobile (mobile/src/)
─────────────────          ──────────────────
api/auth.js      ──100%──→  api/auth.ts      ← 直接搬运
context/AuthCtx  ──90%───→  context/AuthCtx   ← 适配 RN API
pages/*          ──0%────→  app/*             ← 1:1 复刻 UI（重写）
components/*     ──30%───→  components/ui/*   ← 共享组件搬运
utils/*          ──80%───→  utils/*           ← 非 DOM 工具直接搬运
```

## RN 特有约束

| 约束 | 说明 |
|------|------|
| 列表性能 | 大列表必须用 `FlatList` 或 `FlashList`，禁止 `ScrollView` 嵌套大列表 |
| 动画 | 仅使用 `transform` + `opacity`（GPU 加速），禁止动画 `width`/`height` |
| 图片 | 使用 `expo-image`，禁止直接使用 `Image` |
| 安全区域 | 所有屏幕必须处理 `SafeAreaView` |
| 触摸反馈 | 使用 `Pressable`，禁止 `TouchableOpacity` |
| 条件渲染 | 禁止 `&&` 假值渲染，使用三元运算符 |

## 目录结构

```
mobile/
├── app/                    ← Expo Router 页面（文件即路由）
│   ├── (tabs)/             ← Tab 导航
│   ├── auth/               ← 认证页面
│   └── _layout.tsx         ← 根布局
├── src/
│   ├── api/                ← API 封装（从 Web 搬运）
│   ├── components/
│   │   ├── ui/             ← 共享 UI 组件（GlassView, Card, EmptyState...）
│   │   └── business/       ← 业务组件
│   ├── context/            ← React Context（Auth, Language, Toast）
│   ├── hooks/              ← 自定义 Hooks
│   ├── utils/              ← 工具函数
│   └── constants/          ← 常量
└── assets/                 ← 图片/字体
```
