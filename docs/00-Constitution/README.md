## Agent 治理

Constitution 对所有 Agent 具有最终权威。所有 Agent 在执行任务前必须检查：

1. PM Agent → 检查产品信任原则、技术约束
2. Architect Agent → 检查技术约束、安全策略
3. Backend Agent → 检查安全策略、数据库变更铁律、测试铁律
4. Frontend/Mobile Agent → 检查编码规范、安全策略
5. QA Agent → 检查测试铁律
6. DevOps Agent → 检查安全策略、数据库变更铁律

## 核心原则

1. **信任优先于功能** — 产品问题可以改，信任问题很难修
2. **数据安全第一** — 出错前有备份，出错时有预案，出错后快恢复
3. **诚实透明** — 真正透支品牌的是掩盖和无能，而非故障本身
4. **渐进式交付** — 按模块、按优先级逐步上线，避免"一股脑"交付
5. **文档驱动** — Agents communicate through documents, not chat
6. **Constitution 至上** — 所有决策必须以 Constitution 为准
