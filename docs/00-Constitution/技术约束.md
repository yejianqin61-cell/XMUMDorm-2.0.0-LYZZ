# 技术约束

## 技术栈（硬约束）

| 层面 | 技术 | 版本 | 约束 |
|------|------|------|------|
| 后端运行时 | Node.js + Express | — | 不可切换框架 |
| 数据库 | MySQL | — | `jack_campus` 库，使用 `database.js` 的 `query` 方法 |
| 鉴权 | JWT | — | 中间件 `middleware/auth.js` |
| Web 前端 | React | 18+ | 不可引入 Next.js / Remix 等 SSR 框架（已有独立 SSR 方案） |
| 移动端 | React Native (Expo) | SDK 52+ | `mobile/` 目录，Expo Router 文件系统路由 |
| 状态管理 | TanStack Query | — | Web + RN 统一使用 |
| 样式体系 | 液态玻璃设计 | — | CSS 模式 → `expo-blur`（RN） |
| 图片存储 | 本地存储 (Multer) | — | 非云存储；类型 jpg/png/webp，≤ 5MB |
| 通知推送 | Web Push + RN Push | — | 双通道推送 |

## 不可变约束

1. **API 层 100% 复用** — 后端 API 不因 Web/App 而分叉
2. **单库多端** — 同一仓库 `XMUMDorm-2.0.0-LYZZ`，Web 在 `frontend/`，RN 在 `mobile/`
3. **路由文件命名** — 后端 `routes/<模块名>.js`，前端 `pages/<PageName>.jsx`
4. **迁移系统** — 使用 `migrations/` + `scripts/run-incremental-migrations.js`，不可手动改库
5. **用户角色** — `student | merchant | admin`（admin 即官方号）
