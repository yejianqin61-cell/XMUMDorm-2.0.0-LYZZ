# 测试铁律

## 核心原则

> **新增后端路由必须写测试。关键路径必须有集成测试。Middleware 目标 100% 覆盖。**

---

## 测试矩阵

| 层级 | 测试类型 | 框架 | 要求 |
|------|----------|------|------|
| Middleware | 单元测试 | Jest | **目标 100%** 覆盖 |
| Routes | 集成测试 | Jest + Supertest | **新路由必须测试** |
| Services | 单元测试 | Jest | 纯逻辑函数必测 |
| Utils | 单元测试 | Jest | 工具函数必测 |
| 前端组件 | 快照/交互测试 | Jest + Testing Library | 核心组件有测试 |
| E2E | 端到端 | — | 关键流程（登录/发帖/食堂）手动验证 |

## 覆盖率目标

| 文件类型 | Statements | Branches | Functions | Lines |
|----------|-----------|----------|-----------|-------|
| Middleware | ≥ 95% | ≥ 85% | 100% | 100% |
| Services | ≥ 80% | ≥ 70% | ≥ 90% | ≥ 80% |
| Routes | ≥ 75% | ≥ 60% | ≥ 80% | ≥ 75% |
| Utils | ≥ 90% | ≥ 80% | 100% | ≥ 90% |

## 必须测试的场景

### 所有 Route
- [ ] 正常请求返回 200 + 正确数据结构
- [ ] 未登录请求返回 401
- [ ] 参数缺失返回 400
- [ ] 资源不存在返回 404
- [ ] 无权限操作返回 403
- [ ] 数据库错误返回 500

### 所有 Middleware
- [ ] 正常放行
- [ ] 异常拦截
- [ ] 边界条件（空值、无效 token 等）

### Auth 相关
- [ ] 登录成功返回 JWT
- [ ] 错误密码返回 401
- [ ] Token 过期处理

## 运行测试

```bash
# 运行全部测试
npm test

# 运行单个文件
npx jest __tests__/routes/auth.test.js

# 带覆盖率
npm test -- --coverage

# 监听模式
npx jest --watch
```

## 禁止事项

- ❌ 禁止跳过测试提交新路由
- ❌ 禁止使用 `test.skip` 绕过失败测试（除非有明确原因 + TODO）
- ❌ 禁止在测试中硬编码生产环境密钥
- ❌ 禁止测试依赖执行顺序（每个测试独立）
