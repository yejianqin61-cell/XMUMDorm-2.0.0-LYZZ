# 帖子卡片 · 话题/标签（ASCII 方案）

后续「话题专题」可按下述结构扩展；当前前端仅占位展示。

## 1) 卡片 UI 线框（目标布局）

```
+------------------------------------------+
|  [#树洞]  [可选第二标签]                  |  <- 标签栏（替代原头像昵称）
|                                          |
|  正文摘要最多三行……                       |
|                                          |
|  [缩略图]                                 |
|                                          |
|  2小时前 · 💬 3                          |  <- 时间与评论数（轻量）
+------------------------------------------+  <- 分隔（约下 1/4 顶边）
|  (○) 用户昵称              [  ♥ 12  ]   |  <- 底栏：左作者，右大点赞
+------------------------------------------+
```

## 2) 数据模型（建议，未落库前可前端占位）

```
post
├── id
├── content
├── type                 -- normal | announcement（公告可映射标签「公告」）
├── topic_ids[]          -- 未来：关联 topic 表
├── topics[]             -- 未来：列表接口直接带 { id, slug, name } 便于渲染
└── ...
```

## 3) 话题表（未来）

```
topic
├── id
├── slug                 -- URL 用，如 daily-life
├── name                 -- 展示名，如 日常
├── description          -- 专题页简介（可选）
└── sort_order
```

## 4) 路由与专题页（未来）

```
/treehole                    全部
/topic/:slug                 某话题下帖子列表（复用列表组件 + topic 筛选）
```

## 5) 列表接口查询参数（未来）

```
GET /api/posts?topic=slug
GET /api/posts?topic_id=1
```

## 6) 当前占位策略

- `type === 'announcement'` → 标签「公告」
- 否则 → 标签「树洞」（或 `post.topics[0].name` 若已有）
- 无 `topics` 字段时不报错
