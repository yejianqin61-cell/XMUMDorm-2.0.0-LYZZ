## 二、等级系统 (Level) — 步骤 9

### 2.1 性质说明

等级系统不是独立模块，而是**横切关注点**。EXP 已在后端授予（登录/点赞/评论/发帖/点评），移动端需要做的是：

1. **显示**：在个人页展示等级徽章 + EXP 进度条
2. **反馈**：在 EXP 相关操作后显示 Toast 和升级弹窗
3. **说明**：提供等级体系说明页

### 2.2 改动清单

#### A. 增强 `MyZoneScreen.tsx`（已有→扩展）

已在 MyZoneScreen 中显示用户信息（头像/昵称/等级文字），需新增：
- **等级徽章**：emoji + "Lv.N" 格式（如 🌱 Lv.1）
- **EXP 进度条**：当前等级 EXP 进度（如 "45/100 XP"），渐变色进度条

#### B. 新增 EXP 反馈上下文 `ExpFeedbackContext.tsx`

```
功能：
- handleExpResponse(result) — 解析 API 返回的 __exp 字段
- 显示 Toast：绿色 "+N 经验" 或红色 "-N 经验"
- 显示 LevelUpModal：升级时弹出祝贺弹窗（动画效果）
```

#### C. 新增 `LevelUpModal.tsx` 组件

升级弹窗：
- 半透明蒙层 + 居中卡片
- 大号等级 emoji + "恭喜升级 Lv.N!"
- 新徽章名称（如"探索者"）
- 关闭按钮

#### D. 新增 `AboutLevelScreen.tsx` 等级说明页

静态信息页（双语）：
- 等级表：Lv.1-6 对应的 EXP 范围和徽章
- EXP 获取规则表：每个行为的 EXP 值和每日上限
- 优质评价加成、热门帖子奖励说明

#### E. 接入现有 EXP 操作点

在以下已有屏幕的操作回调中加入 `handleExpResponse`：
- `LoginScreen.tsx` — 登录成功
- `TreeholeScreen.tsx` — 点赞/评论/发帖

## 三、一站通 (Handbook) + 课程评价 — 步骤 10-11

> 一站通和课程评价共享后端路由 `/api/handbook`，合并为一个步骤实现。

### 3.1 移动端屏幕设计

采用状态机导航模式，枢纽文件 `HandbookScreen.tsx`：

```
ViewState =
  | { screen: 'home' }
  | { screen: 'articleDetail'; articleId: number }
  | { screen: 'editor'; editId?: number }
  | { screen: 'me' }
  | { screen: 'courseList' }
  | { screen: 'courseDetail'; reviewId: number }
  | { screen: 'courseCreate'; editId?: number }
```

#### 屏幕清单

| # | 文件 | 功能 | 核心交互 |
|---|------|------|---------|
| 1 | `HandbookScreen.tsx` | 状态机导航枢纽 | 7 视图切换 |
| 2 | `HandbookHomeScreen.tsx` | 一站通首页 | 标签切换（美食/游玩/榴莲/校园/学法）+ 搜索 + 文章卡片列表 + 无限滚动 |
| 3 | `HandbookArticleDetailScreen.tsx` | 文章详情 | Markdown 渲染 + 点赞/收藏（乐观更新）+ 评论区（嵌套回复） |
| 4 | `HandbookEditorScreen.tsx` | 创作/编辑文章 | 标题+摘要+Markdown内容+封面+标签选择+发布/草稿 |
| 5 | `HandbookMeScreen.tsx` | 个人中心 | 双Tab：已收藏文章 + 我的课程评价 |
| 6 | `CourseReviewListScreen.tsx` | 课程评价列表 | 搜索（课程名/教师）+ 标签过滤（MPU/GE/ME/必修/期末/无期末）+ 评分卡片列表 |
| 7 | `CourseReviewDetailScreen.tsx` | 评价详情 | 星级展示 + 难度 + 学期 + 匿名评论区 + 星级评分（1-5） |
| 8 | `CourseReviewCreateScreen.tsx` | 创建/编辑评价 | 课程名+教师+学期+标签+评论+评分+难度（编辑模式复用） |

#### 屏幕精简策略
- **Markdown 简化**：文章详情使用 `react-native-markdown-display` 或基础渲染（标题/段落/列表/图片），不做完整 Markdown
- **清单功能省略**：个人 Checklist 功能（低频使用，V2 再迁移）
- **分享功能简化**：分享计数在移动端用"复制链接"代替 Web 分享 API
- **图片上传简化**：暂不支持 Markdown 内嵌图片上传，使用外链图片

## 六、关键技术决策

### 6.1 等级系统
- **EXP 反馈独立上下文**：创建 `ExpFeedbackContext.tsx`，与 AuthContext 解耦
- **API 响应解析**：现有 `request.js` 已自动提取 `__exp` 字段，移动端 `client.ts` 需要同样处理
- **client.ts 增强**：在 `apiPost`/`apiPatch` 中自动提取并透传 `exp` 字段

### 6.2 一站通文章
- **Markdown 渲染**：使用 `react-native-markdown-display`（纯 JS，无需原生模块）
- **文章编辑器**：简化版，仅支持纯文本 Markdown（不内置图片工具栏）
- **TOC 导航**：详情页顶部显示目录下拉（从标题解析）

### 6.3 课程评价
- **匿名设计**：与 Web 端一致，作者信息不暴露
- **标签多选**：6 个预设标签，支持多选（OR 逻辑）
- **学期选择器**：年份 2016-2026 + 月份 02/04/09
