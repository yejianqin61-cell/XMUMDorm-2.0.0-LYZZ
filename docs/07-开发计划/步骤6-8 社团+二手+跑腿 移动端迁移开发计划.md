## 二、社团系统 (Clubs) — 步骤 6

### 2.1 移动端屏幕设计

采用状态机导航模式（同 EatScreen / SquareScreen），枢纽文件 `ClubsScreen.tsx`：

```
ViewState =
  | { screen: 'home' }
  | { screen: 'list' }                    // 社团大全
  | { screen: 'profile'; clubId: number } // 社团详情
  | { screen: 'members'; clubId: number } // 全部成员
  | { screen: 'myClubs' }                 // 我的社团
  | { screen: 'activityDetail'; id: number }  // 活动详情
  | { screen: 'postDetail'; id: number }      // 帖子详情
  | { screen: 'createPost'; clubId: number; type: 'activity'|'post' }
```

#### 屏幕清单

| # | 文件 | 功能 | 核心交互 |
|---|------|------|---------|
| 1 | `ClubsScreen.tsx` | 状态机导航枢纽 | 8 个视图切换 |
| 2 | `ClubsHomeScreen.tsx` | 社团广场主页 | 类别过滤（6标签）+ 动态流（活动/帖子混合）+ 搜索入口 |
| 3 | `ClubProfileScreen.tsx` | 社团详情页 | 关注/取消关注 + 编辑资料（管理员）+ 成员预览 + 活动/帖子列表 + 发布入口 |
| 4 | `ClubListScreen.tsx` | 社团大全 | 搜索 + 类别过滤 + FlatList 网格 |
| 5 | `MyClubsScreen.tsx` | 我的社团 | 已加入社团列表 |
| 6 | `ActivityPostDetailScreen.tsx` | 活动/帖子详情（共用） | 点赞切换 + 评论树 + 回复 + 浏览记录 |
| 7 | `PublishClubContentScreen.tsx` | 发布活动/帖子（共用） | 标题 + 内容 + 图片（最多4张）+ 活动额外字段（时间/地点/报名链接） |
| 8 | `ClubMembersScreen.tsx` | 全部成员列表 | FlatList + 角色标识 |

#### 屏幕精简策略
- **合并** ActivityDetail + ClubPostDetail → `ActivityPostDetailScreen`（通过 `targetType` 区分）
- **合并** PublishActivity + PublishClubPost → `PublishClubContentScreen`（通过 `contentType` 区分）
- **省略** CreateClub（仅站点管理员，移动端低频）
- **省略** ClubCommentsSection 独立组件 → 内嵌到详情页底部评论区

POST /api/clubs/likes/toggle      — 点赞切换
POST /api/clubs/views             — 记录浏览
POST /api/clubs/:id/activities    — 发布活动
POST /api/clubs/:id/posts         — 发布帖子

## 三、二手市场 (Marketplace) — 步骤 7

### 3.1 移动端屏幕设计

状态机导航枢纽 `MarketplaceScreen.tsx`：

```
ViewState =
  | { screen: 'home' }
  | { screen: 'detail'; itemId: number }
  | { screen: 'publish'; editId?: number }
  | { screen: 'myWants' }
  | { screen: 'chat'; threadId: number }
  | { screen: 'chatList'; itemId: number }
```

#### 屏幕清单

| # | 文件 | 功能 | 核心交互 |
|---|------|------|---------|
| 1 | `MarketplaceScreen.tsx` | 状态机导航枢纽 | 6 个视图切换 |
| 2 | `MarketplaceHomeScreen.tsx` | 二手首页 | 分类标签栏 + 状态过滤（在售/已售）+ 搜索 + 价格筛选抽屉 + 商品卡片列表 |
| 3 | `MarketplaceDetailScreen.tsx` | 商品详情 | 图片轮播 + 收藏切换 + 卖家信息 + 聊天入口 + 状态切换（卖家）+ 编辑/删除（卖家） |
| 4 | `MarketplacePublishScreen.tsx` | 发布/编辑商品 | 标题+描述+价格+分类+配送方式+宿舍区域+标签+图片（最多4张）+ 编辑模式复用 |
| 5 | `MarketplaceMyWantsScreen.tsx` | 我的收藏 | 双列网格 + 空状态 |
| 6 | `MarketplaceChatScreen.tsx` | 聊天界面 | 消息列表（4s 轮询）+ 输入框 + 已读标记 + 商品上下文 |

#### 屏幕精简策略
- **合并** create + edit → `MarketplacePublishScreen`（通过 `editId` 参数区分）
- **合并** chat detail + chat threads → `MarketplaceChatScreen`（通过 `threadId` 或 `itemId` 参数区分模式）

### 3.3 聊天实现方案

移动端采用与 Web 一致的**轮询方案**（每 4 秒）：
- FlatList 自动滚动到底部
- `refetchInterval: 4000` 实现消息刷新
- 已读标记在进入页面和发送消息后自动触发

## 七、风险与注意事项

1. **社团模块体量大**：13 个 Web 页面精简为 8 个移动端屏幕，需要仔细设计信息架构，避免遗漏核心功能
2. **二手聊天系统**：Web 端用轮询，移动端也用轮询（4s）；未来可升级为 WebSocket
3. **跑腿 `taken_by_user_id` 始终为 NULL**：后端 BUG，但不影响移动端功能（接单/完成逻辑由所有者控制）
4. **图片上传**：三个模块都需要图片上传，需确认 `expo-image-picker` 已正确集成
5. **FormData 上传**：社团活动和帖子、二手商品、跑腿任务均需 FormData 上传，与现有树洞发帖逻辑一致
6. **中间件链路**：所有发布操作经过 `authenticateToken → checkSanction → sensitiveWordFilter`
