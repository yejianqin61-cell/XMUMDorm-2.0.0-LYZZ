# 通知系统模块化改进开发计划（V3.1）

**依据文档：**
- `docs/03-技术参考/全站用户互动行为清单.md`

**项目：** Dorm · XMUMDorm-2.0.0-LYZZ  
**编写目的：** 将通知系统从按行为分类重构为按模块分类，补全全网互动通知缺口。

## 1. 通知类型新体系

### 1.1 命名规范

统一采用 `{module}_{action}` 格式：

```
treehole_like          — 树洞点赞
treehole_comment       — 树洞评论

trending_like          — 热搜点赞
trending_comment       — 热搜评论

campus_like            — 校园此刻点赞
campus_comment         — 校园此刻评论

canteen_review         — 食堂点评
canteen_reply          — 食堂回复

marketplace_want       — 二手收藏
marketplace_chat       — 二手聊天

club_follow            — 社团关注
club_like              — 社团点赞
club_comment           — 社团评论

system_announcement    — 系统公告
system_ban             — 账号封禁
```

### 1.2 模块分组（前端 Tab）

## 2. 后端改动

### 2.1 通知工具服务（新增）

**新建文件：** `services/notificationService.js`

```js
// 统一通知写入函数
async function createNotification({ userId, type, fromUserId, postId, commentId, extra }) {
  const { query } = require('../database');
  await query(
    'INSERT INTO notifications (user_id, type, post_id, comment_id, from_user_id, extra) VALUES (?, ?, ?, ?, ?, ?)',
    [userId, type, postId || null, commentId || null, fromUserId || null, extra ? JSON.stringify(extra) : null]
  );
}
```

提供统一的 `createNotification()` 函数，所有路由文件通过它写入通知，确保格式一致。

### 2.2 逐模块埋点

#### 树洞（`routes/posts.js`）— 已有，重命名

| 行为 | 旧 type | 新 type | 接收者 |

| 评论 | `comment` | `treehole_comment` | 帖主 |
| 公告 | `announcement` | `system_announcement` | 全站 |

**改动：** 替换 2 处 `INSERT INTO notifications` 的 type 值。

#### 校园此刻（`routes/square.js`）— 新增 3 处

#### 社团（`routes/clubs.js`）— 新增 6 处

| 点赞内容 | `club_like` | 社团所有管理员 | `POST /likes/toggle` |
| 评论活动 | `club_comment` | 社团所有管理员 | `POST /activity/:id/comments` |
| 回复活动评论 | `club_comment` | 评论发布者 | 同上（parent_id 不为空） |
| 评论帖子 | `club_comment` | 社团所有管理员 | `POST /post/:id/comments` |
| 回复帖子评论 | `club_comment` | 评论发布者 | 同上（parent_id 不为空） |

**注意：** 社团管理员列表通过 `club_members WHERE role='admin'` 查询，需向每个管理员各写一条通知。

#### 二手市场（`routes/marketplace.js`）— 已有，重命名

| 行为 | 旧 type | 新 type | 接收者 |
|------|---------|---------|--------|
| 收藏 | `marketplace` | `marketplace_want` | 卖家 |

#### 系统通知（`routes/admin.js`）— 新增 1 处

| 行为 | 新 type | 接收者 | 插入位置 |

| 封禁用户 | `system_ban` | 被封用户 | `POST /users/:id/ban` |
| 发布公告 | `system_announcement` | 全站 | `POST /announcements`（替换旧 `announcement`） |

### 2.3 通知列表 API 增强

**文件：** `routes/notifications.js`

**`GET /` 改动：**
- 支持 `?module=` 参数，映射到一组 type（如 `module=treehole` → `type IN ('treehole_like','treehole_comment')`）
- 保持 `?type=` 兼容

**`GET /unread-summary` 改动：**
```js
// 按模块分组计数
const modules = {
  treehole: ['treehole_like', 'treehole_comment'],
  trending: ['trending_like', 'trending_comment', 'campus_like', 'campus_comment'],
  canteen: ['canteen_review', 'canteen_reply'],
  marketplace: ['marketplace_want', 'marketplace_chat'],
  club: ['club_follow', 'club_like', 'club_comment'],
  system: ['system_announcement', 'system_ban'],
};
// 返回 { byModule: { treehole: 3, trending: 0, ... }, total: N }
```

**`DELETE /clear` 改动：**
- 支持 `?module=` 参数，按模块清除

### 2.4 attachNotificationExtra 增强

为每种新 type 提供正确的 `target` 跳转路径：

| type | target.path |
|------|-------------|
| `treehole_like` / `treehole_comment` | `/post/{post_id}` |
| `trending_like` / `trending_comment` | `/about/trending/post/{post_id}` |
| `campus_like` / `campus_comment` | `/about/campus/{post_id}` |
| `canteen_review` / `canteen_reply` | `/eat/food/{product_id}`（通过 extra.targetPath） |
| `marketplace_want` / `marketplace_chat` | `/about/second-hand/item/{id}` |
| `club_follow` / `club_like` / `club_comment` | `/about/club/{id}` 或 `/about/club/activity/{id}` |
| `system_announcement` | `/post/{post_id}` |
| `system_ban` | —（无跳转） |
