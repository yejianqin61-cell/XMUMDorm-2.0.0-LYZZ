## 一、总体概述

| 模块 | Web 页面数 | API 端点数 | 用户群 | 复杂度 |

| 举报系统 (Report) | 2 页面 + 1 组件 | 4 | 普通用户 + 管理员 | 🟡 中 |

### 关键发现

1. **举报系统已全面嵌入各模块**：Web 端 8 个页面引用了 `ReportButton` 组件，移动端需要在对应屏幕中补齐
2. **管理员后台处理举报的 action（隐藏/删除/禁言/封禁）后端未实际执行**：仅更新了 report 状态（已确认的 bug）
3. **自动阈值仅 `report_auto_review_threshold` 生效**：`auto_hide` 和 `auto_delist` 有配置但无代码实现

## 四、移动端特有设计

### 4.1 管理员入口保护
- 仅 `user.role === 'admin'` 时 MyZone 显示「管理员后台」入口
- AdminScreen 挂载时二次校验（API 请求失败 → 退回 MyZone）
- 所有 admin API 调用依赖后端 `requireAdmin` 中间件（服务端强制）

### 4.2 管理员导航
- 顶部栏显示当前模块名称
- 左上角返回按钮回到 MyZone
- 右上角汉堡菜单 → 弹出模块列表（等同于 Web 侧边栏）
- 模块列表：
  ```
  📊 数据面板    👥 用户管理    🚩 举报中心
  📦 内容管理    📢 公告管理    🛡️ 敏感词
  ⚙️ 系统配置    📋 操作日志
  ```

### 4.3 用户制裁（封禁/禁言）
- UserDetail 页底部显示操作按钮
- 封禁/禁言弹出时长选择器（Web `UserActionModal` 的 RN 版本）：
  - 时长选项：1天 / 3天 / 7天 / 30天 / 永久
  - 可选原因输入
  - 确认后调用 API

### 4.4 内容管理模块选择
- 下拉或网格选择 9 个内容模块之一
- 选中后显示该模块的内容列表
