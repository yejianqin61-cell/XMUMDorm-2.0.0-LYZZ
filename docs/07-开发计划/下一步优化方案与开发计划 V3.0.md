### 2.6 改进导航返回栈

**问题**: 深层导航（如 广场 → 热搜 → 话题 → 帖子详情）按"返回"直接回到 Tab 首页而非上一级。

**方案**: 在状态机中增加历史栈：

```tsx
// SquareScreen 为例
const [history, setHistory] = useState<ViewState[]>([]);

function pushView(v: ViewState) {
  setHistory(prev => [...prev, view]);
  setView(v);
}

function popView() {
  if (history.length > 0) {
    const prev = history[history.length - 1];
    setHistory(history.slice(0, -1));
    setView(prev);
  } else {
    setView({ screen: 'home' });
  }
}
```

| 任务 | 影响范围 | 工时 |
|------|---------|------|
| 为 5 个状态机 Screen 增加历史栈 | Square / Clubs / Marketplace / Handbook / Admin | 3h |

## 三、Phase 2 — 增强（Enhancement）

**目标**: 补齐功能缺口，提升代码质量和用户体验。

### 3.1 后端 FCM 推送通道实现

**现状**: 移动端已注册 FCM token 到服务端，但服务端 `services/pushSend.js` 只支持 Web Push（VAPID）。需要新增 FCM 发送通道。

**方案**:

```
后端新增:
├── services/fcmSend.js         ← Firebase Admin SDK 封装
├── services/pushSend.js 编辑   ← 增加 fcm channel 分支
├── routes/push.js 编辑         ← subscribe 增加 channel 字段
├── migrations/                ← push_subscriptions 表增加 channel 列
└── server.js 编辑              ← 初始化 firebase-admin

移动端 (已完成):
└── src/services/pushService.ts ← FCM token 注册/注销
```

|------|------|
| 后端 `fcmSend.js` + Firebase Admin 初始化 | 2h |
