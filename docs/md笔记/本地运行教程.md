# 把电脑当服务器：本地运行 Jack 网站（含食堂后端）

> 按这个教程做下来，你的电脑就是「服务器」，可以在本机打开网站、使用后端接口；**默认只能在你电脑上或同一 WiFi 下的设备访问**（不暴露到公网）。

---

## 一、你的理解对不对？

**对的。**

- **本地迁移数据库** = 在你电脑的 MySQL 里执行建表脚本，把「区域、店铺、商品、评论」等表建好。
- **在本机运行后端** = 用 `npm start` 启动 Node 服务，你的电脑就在提供接口（API）。
- **仅限于本地** = 服务默认监听 `http://127.0.0.1:4040`，只有你这台电脑（或同一局域网）能访问；不部署到云的话，外网访问不到。

---

## 二、事前准备（只做一次）

### 1. 安装 Node.js

- 打开 [https://nodejs.org/](https://nodejs.org/) 下载并安装（选 LTS 即可）。
- 装好后打开 **命令提示符** 或 **PowerShell**，输入：
  ```bash
  node --version
  npm --version
  ```
  能显示版本号就说明装好了。

### 2. 安装并启动 MySQL

- 若还没装 MySQL，可装 [MySQL Community Server](https://dev.mysql.com/downloads/mysql/) 或 [XAMPP](https://www.apachefriends.org/)（里带 MySQL）。
- 确保 **MySQL 服务已启动**（XAMPP 的话在控制面板里点 MySQL 的 Start）。
- 记下你的 **MySQL 用户名**（一般是 `root`）和 **密码**（安装时设的；没设则可能为空）。

### 3. 项目代码

- 确保 Jack 项目已经在你电脑上（例如 `C:\Users\你的用户名\Desktop\Jack`）。
- 后面的命令都要在 **项目根目录** 下执行（即能看到 `package.json`、`server.js` 的那一层）。

---

## 三、一步步操作

### 第 1 步：安装依赖

在项目根目录打开终端（命令提示符或 PowerShell），执行：

```bash
cd C:\Users\叶健钦\Desktop\Jack
npm install
```

（如果当前已经在 Jack 目录，只执行 `npm install` 即可。）

---

### 第 2 步：准备数据库（建库 + 建表）

需要让 MySQL 执行两个 SQL 文件（顺序不要反）：

**2.1 初始化数据库和用户、帖子等表**

在终端执行（把 `你的MySQL密码` 换成你的 root 密码；没密码就改成 `-p` 后面不写东西，回车后再输密码或直接回车）：

```bash
mysql -u root -p你的MySQL密码 < init-db.sql
```

- 若提示 `mysql 不是内部或外部命令`：说明 MySQL 没加到环境变量。可以：
  - 用 **完整路径**，例如：`"C:\Program Files\MySQL\MySQL Server 8.0\bin\mysql" -u root -p < init-db.sql`
  - 或先打开 **MySQL 命令行 / MySQL Workbench**，在里面用 `source` 执行：  
    `source C:/Users/叶健钦/Desktop/Jack/init-db.sql`
- 执行成功后会创建 `jack_campus` 数据库，以及用户表、帖子相关表等。

**2.2 食堂系统表（区域、店铺、商品、评论等）**

再执行：

```bash
mysql -u root -p你的MySQL密码 jack_campus < migrations/002_canteen_system.sql
```

- 同样，如果 `mysql` 找不到，用上面的完整路径或 MySQL 客户端里的 `source` 执行：  
  `source C:/Users/叶健钦/Desktop/Jack/migrations/002_canteen_system.sql`
- 执行成功后，数据库里会有区域、店铺、分类、商品、评论等表，食堂接口才能用。

---

### 第 3 步：配置环境变量（让后端连上数据库）

在项目根目录（和 `server.js` 同级）新建一个文件，命名为 **`.env`**（没有名字，只有后缀），内容示例：

```env
PORT=4040
NODE_ENV=development
JWT_SECRET=随便写一串别人猜不到的字符串

DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=你的MySQL密码
DB_NAME=jack_campus
```

- `DB_PASSWORD` 改成你 MySQL 的 root 密码；没密码就留空：`DB_PASSWORD=`
- 保存后，后端会用这些配置连你本机的 MySQL。

---

### 第 4 步：启动后端服务（把电脑当服务器）

在项目根目录执行：

```bash
npm start
```

看到类似输出就说明成功：

```
========================================
🚀 服务器已启动！
📍 地址: http://127.0.0.1:4040
🌍 环境: development
✅ 数据库连接成功！
========================================
```

此时：

- **你的电脑 = 服务器**
- 后端服务只在你本机（和同一局域网）可访问，**仅限于本地/局域网**。

---

### 第 5 步：验证是否成功

1. **测根地址**  
   浏览器打开：`http://127.0.0.1:4040/`  
   应看到类似：`{"message":"Jack 校园社交网站后端服务运行正常！","version":"1.0.0"}`

2. **测食堂区域接口**  
   打开：`http://127.0.0.1:4040/api/canteen/regions`  
   应返回 5 个区域（D6、LY3、B1、BELL、other）的 JSON。

3. **用前端页面（若已配置 API 地址）**  
   打开项目里的 `html/login.html` 或 `html/register.html`；若里面的 `config.js` 里 API 地址是 `http://127.0.0.1:4040`，就可以在本机登录、注册，使用后端。

---

## 四、以后每次想在本机运行网站

1. 确保 **MySQL 服务已启动**。
2. 在项目根目录执行：`npm start`。
3. 浏览器访问 `http://127.0.0.1:4040` 或你的前端页面。

**不需要再执行迁移**：建表只需做一次（第 2 步）；以后除非你删库或换电脑，否则不用重复执行 `init-db.sql` 和 `002_canteen_system.sql`。

---

## 五、常见问题

| 情况 | 处理 |
|------|------|
| `mysql` 找不到 | 用 MySQL 的完整路径，或在 MySQL 客户端里用 `source 脚本路径` 执行。 |
| 数据库连接失败 | 检查 `.env` 里 `DB_PASSWORD`、`DB_USER`、`DB_HOST` 是否和本机 MySQL 一致；MySQL 服务是否已启动。 |
| 端口 4040 被占用 | 在 `.env` 里改 `PORT=4041`（或别的未占用端口），再 `npm start`。 |
| 食堂接口 500 / 报错 | 确认已执行过 `002_canteen_system.sql`（第 2.2 步）。 |

---

**总结**：  
本地迁移 = 在本机 MySQL 里建好表；然后 `npm start` = 用你的电脑当服务器跑后端。这样就在**本地**拥有了完整后端服务，仅限于你电脑（或同一局域网）访问。
