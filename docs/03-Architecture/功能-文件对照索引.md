# Jack 网站：功能行为与文件对照（功能驱动）

> 按「一个行为一条数据流」整理：每个行为用到的页面、前端文件、后端路由与中间件、数据库。便于按功能逐条理解数据流。

---

## 一、认证与登录态

### 1.1 用户注册（学生）

| 项目 | 说明 |
|------|------|
| **行为** | 用户填写邮箱(@xmu.edu.my)、用户名、密码（可选验证码），提交后创建学生账号并自动登录。 |
| **涉及页面** | 注册页 `/register` |
| **数据流** | 表单提交 → `POST /api/auth/register` → 校验角色/邮箱/用户名/密码 → 写 `users` 表 → 生成 JWT 并返回 → 前端存 token/user 到 localStorage，跳转首页。 |

**前端文件：**

| 文件 | 作用 |
|------|------|
| `frontend/src/App.jsx` | 路由 `path="/register"` → `<Register />` |
| `frontend/src/pages/Register.jsx` | 表单、校验、`fetch(API_BASE_URL + '/api/auth/register')`、写 localStorage、跳转 |
| `frontend/src/api/config.js` | `API_BASE_URL` |

**后端文件：**

| 文件 | 作用 |
|------|------|
| `server.js` | 挂载 `app.use('/api/auth', authRoutes)` |
| `routes/auth.js` | `POST /register`：校验 role/email/username/password、bcrypt 加密、`query('INSERT INTO users ...')`、jwt.sign、返回 token+data |
| `database.js` | `query()` 执行 SQL |
| 表 | `users`（username, email, password_hash, role, email_verified） |

### 1.2 用户登录（学号/邮箱 + 密码）

| 项目 | 说明 |
|------|------|
| **行为** | 用户输入学号或邮箱、密码，提交后校验，返回 JWT；前端保存登录态并跳转。 |
| **涉及页面** | 登录页 `/login` |
| **数据流** | 表单提交 → AuthContext.login() → `POST /api/auth/login` → 查 users、校验密码 → 生成 JWT → 前端存 token/user，跳转 from 或首页。 |

**前端文件：**

| 文件 | 作用 |
|------|------|
| `frontend/src/App.jsx` | 路由 `path="/login"` → `<Login />` |
| `frontend/src/pages/Login.jsx` | 表单、调用 `login(sid, password)`、成功后跳转 |
| `frontend/src/context/AuthContext.jsx` | `login()`：`fetch(API_BASE_URL + '/api/auth/login')`，成功后 setToken/setUser、localStorage 存 token/user |
| `frontend/src/api/config.js` | `API_BASE_URL` |

**后端文件：**

| 文件 | 作用 |
|------|------|
| `server.js` | 挂载 `/api/auth` |
| `routes/auth.js` | `POST /login`：按学号或邮箱查 users、bcrypt.compare、jwt.sign、返回 token+data |
| `database.js` | `query()` 查 users |
| 表 | `users` |

### 1.5 路由守卫（未登录且未跳过则跳转登录）

| 项目 | 说明 |
|------|------|
