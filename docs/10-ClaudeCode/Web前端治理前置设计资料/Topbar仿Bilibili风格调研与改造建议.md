# Topbar 仿 Bilibili 风格调研与改造建议

## 调研对象

- Bilibili 首页顶部导航（2026-07-18）：[bilibili.com](https://www.bilibili.com/)
- 当前项目 Topbar：`frontend/src/components/shell/SiteHeader.jsx`
- 已选背景图：`frontend/public/topbar/topbar_background.png`（1506 x 440）

## 可借鉴的规则

1. 顶栏先是完整画面，再是导航。导航不是白底工具条，而是叠在背景图上的前景层。
2. 入口保持等宽、纵向图文结构：线框图标在上，短文字在下。避免胶囊按钮和说明文字。
3. 白色前景需要底部深色遮罩与轻微文字阴影，不能只依赖图片本身保证可读性。
4. 背景图只服务顶栏氛围，不承载可点击内容；小屏设备应降低顶栏高度，保持按钮可触达。

## 当前 Topbar 问题

- 当前白色半透明底和圆角文字按钮，与校园插画背景不匹配。
- 发布、信箱、后台、语言切换的视觉层级不同，不能形成一组清晰的顶部导航。
- 现有头像可保留为账户入口，不应强行改成文字按钮。

## 建议方案

### 结构

- 左侧保留 `XMUMDorm` 品牌文字，但改为白色前景，不增加图标。
- 右侧改为四个紧凑的图文入口：语言、发布、信箱、管理后台（仅管理员）。
- 每个入口使用现有 `lucide-react` 图标，图标上、文字下；发布不再使用实心主按钮。
- 个人头像及姓名继续作为独立账户入口，放在导航组末尾。

### 视觉

- 顶栏高度：桌面端 `104px`，平板端 `88px`，移动端 `72px`。
- 背景：使用 `topbar_background.png`，`background-size: cover`，`background-position: center 56%`。
- 遮罩：从底部的 `rgba(13, 35, 49, 0.58)` 过渡到顶部的 `rgba(13, 35, 49, 0.12)`，保留校园画面同时稳定白字对比度。
- 图标：白色描边，`20px`；文字白色，`12px`，不使用圆角卡片、实心色块或渐变按钮。
- 悬停：仅提高背景透明度并上移 `1px`；键盘焦点使用白色外描边。

### 响应式

- 宽屏显示图标和文字。
- 中等宽度隐藏品牌副标题和用户名。
- 小屏只保留图标，使用 `aria-label` 和原生 tooltip；管理员入口按既有权限条件显示。

## 实施边界

- 只调整 Web 全局 Topbar，不改变路由、权限判断、语言切换、发布、信箱或个人中心的行为。
- 复用当前的 `lucide-react` 与背景图片，不新增依赖或图标资源。

## 待确认

- 默认按上述方案保留左侧 `XMUMDorm` 品牌文字和右侧头像；若希望 Topbar 完全只保留图标，需要在实施前确认。
