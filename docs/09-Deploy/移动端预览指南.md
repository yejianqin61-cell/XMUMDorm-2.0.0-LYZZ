# 手机查看前端效果指南

在电脑上开发时，可以用手机连同一 WiFi 访问项目，实时看真机效果。

---

## 一、前提条件

- 手机和电脑连接**同一个 WiFi**（不能一个有线一个开热点，除非热点和电脑同网段）。
- 电脑上已能正常运行前端：在项目里执行过 `npm install`，且 `npm run dev` 能正常启动。

---

## 二、操作步骤

### 1. 查电脑的 IP 地址（本机局域网 IP）

- **Windows**：按 `Win + R`，输入 `cmd` 回车，在黑色窗口里输入：
  ```text
  ipconfig
  ```
  找到 **“无线局域网适配器 WLAN”** 或 **“以太网”** 下面的 **IPv4 地址**，形如：`192.168.1.105`。
- **Mac**：系统设置 → 网络 → 当前连接（Wi‑Fi 或以太网）→ 详情里可看到 IP；或在终端执行 `ifconfig` 查看。

记下这个 IP，后面在手机浏览器里要用。

### 2. 让开发服务器可被手机访问

本项目已配置 Vite 的 `server.host: true`，所以执行 `npm run dev` 时，服务会同时监听局域网，手机可以通过「电脑 IP + 端口」访问。

在项目根目录的 **frontend** 目录下执行：

```bash
cd frontend
npm run dev
```

终端里会看到类似：

```text
  VITE v5.x.x  ready in xxx ms
  ➜  Local:   http://localhost:5173/
  ➜  Network: http://192.168.1.105:5173/
```

其中 **Network** 那一行的地址就是手机要访问的（端口一般是 `5173`，以你终端里显示的为准）。

### 3. 用手机打开页面

1. 手机连上**和电脑相同的 WiFi**。
2. 打开手机浏览器（Chrome、Safari、微信内置浏览器等均可）。
3. 在地址栏输入：
   ```text
   http://192.168.1.105:5173
   ```
   **把 `192.168.1.105` 换成你在第 1 步查到的电脑 IP**，端口 `5173` 以终端里显示的为准。
4. 回车即可打开当前前端页面，和电脑上看到的是同一套代码，可滑动、点击测试效果。

---

## 三、常见问题

| 现象 | 可能原因 | 处理办法 |
|------|----------|----------|
| 手机打不开页面 | 电脑防火墙拦截 | 在 Windows 防火墙里放行端口 5173，或临时关闭防火墙试一下 |
| 手机打不开页面 | 手机和电脑不在同一 WiFi | 确认两者连的是同一个路由器发出的 WiFi |
| 手机能打开但很卡/白屏 | 资源大或网络慢 | 正常，可多等几秒；或先用电脑确认 `npm run dev` 无报错 |
| 改代码后手机没更新 | 缓存或未保存 | 保存代码后等几秒；或在手机浏览器里下拉刷新、清除缓存后再试 |
| 终端里没有 Network 那一行 | 未配置 host | 确认 `frontend/vite.config.js` 里已有 `server: { host: true }`，然后重新执行 `npm run dev` |

---

## 四、小结

1. 电脑和手机连**同一 WiFi**。  
2. 在 **frontend** 目录执行 **`npm run dev`**，记下终端里 **Network** 的地址（如 `http://192.168.x.x:5173`）。  
3. 手机浏览器输入该地址即可用手机查看效果。  

以后每次想用手机看效果，只要先 `npm run dev`，再用同一 Network 地址在手机打开即可。
