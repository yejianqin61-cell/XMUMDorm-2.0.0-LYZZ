# 前端拆分说明

当前仓库已拆分为两套前端工作区：

- `frontend/`：Web 端工作区，后续可按桌面网页布局继续重构
- `frontend-app/`：App 端工作区，供 Capacitor 构建 APK 使用

后端仍保持共用：

- `server.js`
- `routes/`
- `services/`
- `database.js`

## 当前打包关系

- Web 构建：`npm run build:web`
- App 构建：`npm run build:app`
- Capacitor 构建：`npm run build:capacitor`

其中 `capacitor.config.ts` 已指向：

```ts
webDir: 'frontend-app/dist'
```

这意味着之后即使 `frontend/` 做了桌面化改版，也不会直接影响 APK 打包。

## 开发命令

```bash
# 后端
npm run dev

# Web 前端
npm run dev:web

# App 前端
npm run dev:app
```

## 后续建议

优先在 `frontend/` 中推进网页端改造，把 `frontend-app/` 作为移动端稳定分支使用。

等两边差异逐渐稳定后，再考虑抽取共享代码，例如：

- `api/`
- 请求封装
- 常量
- 纯工具函数
- 部分无布局依赖的业务组件

不建议一开始就强行共享页面层、路由层和整体布局层，否则会重新把 Web 和 App 耦合回去。
