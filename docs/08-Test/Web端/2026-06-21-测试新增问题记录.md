# 2026-06-21 测试新增问题记录

## Issue 1: `TreeHole.jsx` 仍存在较多 React Hook 规范问题

- 严重级别: High
- 影响范围: 树洞首页滚动、图片卡片、分页预取
- 现象:
  - `frontend/src/pages/TreeHole.jsx` 仍有 `react-hooks/set-state-in-effect` 报错
  - 同文件还存在多处 `react-hooks/exhaustive-deps` 警告
- 风险:
  - 高速滚动和图片切换场景下，可能出现额外重渲染、状态抖动或维护成本继续上升
  - 后续继续改树洞时，容易引入新的异步和渲染时序问题
- 建议:
  - 将图片状态同步逻辑从 effect 改成更稳定的状态机或 reducer
  - 拆分无限滚动、副作用预取、图片加载三个职责，降低单文件复杂度

## Issue 2: Web 主包体积仍然偏大

- 严重级别: Medium
- 影响范围: Web 首屏加载、Capacitor 包体
- 现象:
  - `frontend/npm.cmd run build` 产物中主 JS chunk 约 `1.2 MB`
  - Vite 已持续给出 chunk size warning
- 风险:
  - 首屏加载时间偏长
  - 真机弱网环境下体验更差
  - 后续继续堆功能会放大问题
- 建议:
  - 对大页面做路由级 code splitting
  - 评估 `TreeHole`、`Square`、`Canteen` 的重组件按需加载
  - 视情况补 `manualChunks`

## Issue 3: `posts` 点赞接口存在并发竞争窗口

- 严重级别: High
- 影响范围: `POST /api/posts/:id/like`
- 现象:
  - 当前实现先查 `post_likes` 是否存在，再决定 `INSERT` 或 `DELETE`
  - 多次快速点击或多端同时操作时，可能出现重复插入竞争
- 风险:
  - 点赞状态抖动
  - 在唯一索引约束不足或异常处理不完整时，可能出现脏数据或 500
- 建议:
  - 基于唯一索引重构为原子化写法
  - 明确处理 duplicate key，或改为事务/幂等 toggle 方案
