## 1. 已确认产品决策

以下内容已视为本轮开发的确认前提，不再重复讨论：

- 默认首页从树洞切换为广场页
- 底部 Tab 顺序调整为“广场”第一位
- `SquareHome` 承载“今日校园”首页
- 继续采用 `Web + Capacitor` 双形态，不新增 RN 端
- 保持 React 单前端工程演进，当前阶段不处理 React 大版本演进问题
- 发布中心进入开发范围，且属于高优先级
- 社团活动报名闭环属于高优先级
- 广场顶部个性化推荐区属于第二阶段增强，不放入首批上线
- 需关注移动端安全区，尤其顶部状态栏不可遮挡内容

## 2. 优先级总表

| 优先级 | 目标 | 核心内容 |
|------|------|------|
| P0 | 先完成第一眼体验和核心闭环 | 默认首页切换、今日校园首页、统一卡片/状态、基础动效、发布中心、活动报名闭环、通知事务分类 |

| P2 | 最后做校园特色纵深能力 | 新生专区、活动签到回顾、日历联动、完整推荐、关系图谱扩展 |

### P0-Task008 P0 联调、回归与异常处理补齐

**目标**

- 完成 P0 链路联调
- 系统性补齐前端异步状态、`try/catch`、错误提示与兜底逻辑

**影响范围**

- 前端：`SquareHome`、`PublishCenter`、`Mailbox`、`ActivityDetail`

- 后端：聚合接口、报名接口、通知接口的错误响应结构

**依赖**

- 依赖全部 P0 功能开发完成

**建议工时**

- `1d`

**验收标准**

- P0 主流程全部走通

## 7. 联调与验收要求

### 7.1 前端验收重点

- Web 与 Capacitor 双端都能稳定访问核心页面
- 默认首页切换后无导航混乱
- 安全区处理正确，顶部内容不压状态栏
- 公共卡片、空态、错误态、Skeleton 已统一
- 发布、报名、通知等关键交互有明确反馈
- 异步请求失败时页面具备降级能力

### 7.2 后端验收重点

- 新增聚合接口结构稳定
- 报名接口有重复提交与并发保护
- 通知类型扩展不影响旧逻辑
- 推荐接口与行为表设计可扩展
- Express 路由错误返回结构统一

### 7.3 代码质量重点

- 页面级异步逻辑需补足 `try/catch`
- 避免未处理 Promise 异常
- 前端避免多个独立请求阻塞首屏
- 后端聚合逻辑尽量下沉到 service 层，避免路由文件过重
- 统一错误提示、日志记录与空数据兜底

> 先做“默认首页 + 今日校园 + 核心闭环”，再做“个性化和身份感”，最后做“校园特色纵深能力”。
