## 3. 测试子任务 A：P0 首页与导航链路测试

覆盖范围：

- 默认首页切换
- Tab 顺序与高亮
- 广场首页首屏聚合
- 返回路径与深链进入

重点检查：

1. 登录态 / 游客态是否都正确进入广场
2. Tab 切换是否存在高亮错位
3. 首屏接口失败时是否整页白屏
4. Capacitor 顶部安全区是否遮挡内容
5. 页面切换是否存在闪烁、布局跳动

输出物：

- 首页链路缺陷清单
- UI 与布局优化建议

---

## 4. 测试子任务 B：前端组件与状态系统测试

覆盖范围：

- 卡片系统
- 空态 / 错误态 / Skeleton
- 动效反馈

重点检查：

1. 公共卡片在不同页面是否出现样式漂移
2. Skeleton 与真实内容切换时是否闪烁
3. 空态和错误态文案是否清晰
4. 动效是否过重、掉帧或打断操作

输出物：

- UI 一致性问题清单
- 动效性能与节奏建议

---

## 5. 测试子任务 C：前端异步与异常处理专项测试

覆盖范围：

- `SquareHome`
- `PublishCenter`
- `Mailbox`
- `ActivityDetail`
- 其他新接入聚合请求页面

重点检查：

1. 是否存在未处理 Promise 异常
2. `async/await` 是否缺失 `try/catch`
3. 接口超时、断网、空数据、重复点击时页面表现是否合理
4. 局部失败时是否支持分块降级
5. Toast / 错误提示是否可理解

执行建议：

1. 人工断网测试
2. 模拟接口超时 / 500 / 404
3. 连续点击按钮触发重复请求
4. 页面切换中断请求，检查是否出现脏状态回写

输出物：

- 异步错误清单
- `try/catch` 补齐建议
- 页面级容错优化建议

---

## 6. 测试子任务 D：发布中心与活动报名闭环测试

覆盖范围：

- 发布中心
- 活动报名 / 取消报名
- 事务通知回流

重点检查：

1. 发布中心各入口是否都能正确跳转
2. 无权限用户是否误看到受限入口
3. 报名、取消报名、重复报名是否处理正确
4. 报名人数展示是否与数据库一致
5. 事务通知是否生成并能跳转回活动详情

输出物：

- 主闭环问题清单
- 权限与状态一致性问题清单

---

## 7. 测试子任务 E：后端 Express 与接口质量测试

覆盖范围：

- `routes/square.js`
- `routes/clubs.js`
- `routes/notifications.js`
- 相关 service 与 migration

重点检查：

1. 路由层是否堆积过多业务逻辑
2. 错误返回结构是否统一
3. 是否存在缺失参数校验
4. 是否存在重复写库、并发写入、状态竞争问题
5. 聚合接口是否有 N+1 查询或首屏性能风险

执行建议：

1. 针对报名接口做重复请求与并发测试
2. 针对聚合接口测空数据、脏数据、部分模块失败场景
3. 检查 migration 是否可重复执行、是否有回滚风险

输出物：

- 后端潜在 Bug 清单
- Express 结构优化建议
- 接口性能与稳定性建议

---

## 8. 测试子任务 F：P1/P2 增强能力回归测试

覆盖范围：

- 个性化推荐
- 长列表节奏优化
- 个人主页校园身份
- 通知聚合
- 新生专区
- 日历联动

重点检查：

1. 无行为数据时推荐是否合理降级
2. 长列表中插是否破坏主内容滚动体验
3. 校园身份字段是否正确受隐私开关控制
4. 日历联动是否存在时区错位
5. 推荐理由是否可理解

输出物：

- P1/P2 体验问题清单
- 功能拓展建议与优化建议

---

## 9. 缺陷分级建议

### P0 缺陷

- 白屏、崩溃、无法进入首页
- 报名状态错误
- 通知跳转错误
- 权限泄漏
- 未处理异常导致核心链路中断

### P1 缺陷

- 列表错位
- 分类错误
- 部分推荐区展示异常
- 动效卡顿但不影响主流程

### P2 缺陷

- 文案不一致
- 视觉细节问题
- 推荐解释不够清晰

- P0 主链路缺陷全部收敛
- 前端关键页面无明显未处理异步异常
