# T19-主壳响应式退化与移动兼容

## 1. 任务目标

为新的桌面 Web 壳补齐平板、手机端的退化策略，确保桌面优先同时不破坏移动 Web 基本可用性。

## 2. 影响范围

- 主站壳样式
- `Layout / TabBar` 的响应式逻辑
- 桌面与移动壳切换相关代码

## 3. 不可触碰边界

- 不做全站页面的移动细节重写
- 不引入 App 端逻辑

## 4. 实施步骤

1. 明确大桌面、小桌面、平板、手机断点行为。
2. 调整主壳在不同断点下的区域退化。
3. 保留手机 Web 的简化兼容方案。
4. 确保桌面壳与移动兼容逻辑可并存。

## 5. 验收方式

- 大桌面、平板、手机都有明确布局行为
- 手机 Web 兼容未回归
- `npm run build:web` 通过

## 6. 推荐提交信息

`refactor(web-shell): add responsive fallback for desktop shell`
