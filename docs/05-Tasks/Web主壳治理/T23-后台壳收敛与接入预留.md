# T23-后台壳收敛与接入预留

## 1. 任务目标

在不大规模重构后台的前提下，明确后台壳与主站 Step 4 的关系，并完成最小接入预留。

## 2. 影响范围

- `frontend/src/components/Admin/AdminLayout.jsx`
- `frontend/src/pages/SquareOrgAdmin.jsx`
- 后台壳相关样式或文档

## 3. 不可触碰边界

- 不批量重构后台页面
- 不统一后台所有表单、表格和管理逻辑

## 4. 实施步骤

1. 明确后台壳与主站壳的关系。
2. 对齐桌面布局规则和设计语言方向。
3. 完成最小接入预留或最小验证。

## 5. 验收方式

- 后台壳边界清晰
- 与主站壳关系明确
- `npm run build:web` 通过

## 6. 推荐提交信息

`refactor(web-shell): align admin shell with step4 layout rules`
