# Web 主壳治理 Task 总览

## 1. 目标

本目录用于承接 Step 4 Web 主壳替换阶段的可执行 task 文档。

当前这一批 task 基于以下文档拆解：

- `docs/09-Deploy/Step4-Web主壳详细设计文档.md`
- `docs/09-Deploy/Web桌面化布局规范.md`
- `docs/09-Deploy/Web前端迁移步骤与开发顺序说明.md`

目标是把 Step 4 从“设计文档”推进到“可逐个执行、逐个提交、逐个验收”的任务层。

## 2. 执行原则

统一执行规则如下：

1. 一次只做一个 task。
2. 完成一个 task 后单独提交一次。
3. 提交信息必须符合 Conventional Commits。
4. 默认优先改 `frontend/`，不主动改 `frontend-app/` 和 `shared/*`。
5. 优先复用 Step 1 到 Step 3 已经完成的 token、基础组件、页面模板。
6. 壳层 task 不主动混入大批量业务页迁移。

## 3. Task 执行顺序

当前建议严格按下面顺序推进：

1. `T15-主站桌面壳骨架搭建.md`
2. `T16-主站路由壳接入与Tab职责降级.md`
3. `T17-全局头部与左侧导航落地.md`
4. `T18-右侧辅助栏与主内容容器落地.md`
5. `T19-主壳响应式退化与移动兼容.md`
6. `T20-首页主壳验证接入.md`
7. `T21-工作台主壳验证接入.md`
8. `T22-列表页主壳验证接入.md`
9. `T23-后台壳收敛与接入预留.md`

## 4. 推荐提交规范

推荐提交前缀：

- `docs(web-shell): ...`
- `feat(web-shell): ...`
- `refactor(web-shell): ...`
- `chore(web-shell): ...`

## 5. 阶段边界

这一批 task 只覆盖：

- Step 4 Web 主壳替换

暂不覆盖：

- Step 5 高频页面批量迁移
- Step 6 后续模块级治理

## 6. 完成标准

当本目录下全部 task 完成后，意味着：

- Web 主站具备桌面主壳
- 底部 `TabBar` 不再承担桌面主导航职责
- Step 3 页面模板已具备统一落进桌面主壳的入口
- Step 5 高频页面批量迁移具备正式开工条件
