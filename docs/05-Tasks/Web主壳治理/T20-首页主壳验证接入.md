# T20-首页主壳验证接入

## 1. 任务目标

以 `SquareHome` 为首个高频页，验证新 Web 主壳下首页内容入口是否能正常落位。

## 2. 影响范围

- `frontend/src/pages/SquareHome.jsx`
- 主站壳接入相关文件

## 3. 不可触碰边界

- 不在本任务中重做整个广场页面
- 不混入多模块页面迁移

## 4. 实施步骤

1. 让 `SquareHome` 接入新主壳。
2. 调整其在桌面壳中的首屏落位。
3. 验证头部、主区、辅助区关系。
4. 保持业务逻辑不动，优先做结构验证。

## 5. 验收方式

- `SquareHome` 能在主壳中正常显示
- 首页看起来不再像放大的手机页
- `npm run build:web` 通过
