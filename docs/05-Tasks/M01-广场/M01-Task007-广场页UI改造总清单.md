# M01-Task007-广场页UI改造总清单

## 0. 当前收口结论

基于 `M01-Task002` 到 `M01-Task006` 的连续改造，广场首页第一轮 UI 治理已经完成主要收口。

当前可确认：

- 首页结构已经从“详情堆叠页”收敛为“总览导流页”
- 动态推荐区已压缩到 `1~2` 张主卡片
- 热标签、活动、热议已改为更轻的横向浏览结构
- 快捷入口图标语言与发布入口层级已经统一收口
- 发布入口不再以会压迫底部 `TabBar` 的形态出现

这份文档从“改造任务母表”更新为“阶段验收母表”，用于明确本轮已完成项与后续待优化项。

## 1. 改造背景

当前广场页的核心问题不是功能不够，而是首页承载的信息过多，导致：

- 信息密度过高
- 视觉层级混乱
- 页面留白不足
- 模块之间风格不统一
- 首页像“拉不到底的表单”，缺少高级感与节奏感

本次改造目标不是继续加内容，而是对首页做一次明确的“减法重构”，让广场页回到“轻聚合、强导流、可呼吸”的体验方向。

## 2. 改造目标

- 首页只保留最核心的信息感知与高频入口
- 模块之间建立统一的卡片语言和排版节奏
- 明显提升页面留白、阅读舒适度与浏览路径清晰度
- 让用户进入广场后，第一眼知道“先看什么、再点什么”
- 把首页从“全家桶详情页”收敛成“有主次的总览页”

## 3. 总体改造原则

### 3.1 做减法

- 首页不再平铺所有模块详情
- 每个模块默认只展示 `1~2` 条代表性内容
- 可进入二级页的内容，不必在首页完整展开

### 3.2 建立主次

- 首屏只能有一个核心内容焦点
- 其他模块必须弱化为辅助信息或导流入口
- 同屏内避免多个“都很重要”的大卡片并列竞争注意力

### 3.3 统一语言

- 控制卡片种类数量
- 统一圆角、边距、阴影、标题样式、辅助信息样式
- 图标风格统一，避免多套视觉语言混用

### 3.4 强化呼吸感

- 明显增加模块上下间距
- 增加卡片内部留白
- 给标题、正文、辅助信息建立更清晰的字阶层级

## 4. 页面结构改造清单

### 4.1 首屏结构重排

- [x] 顶部 Banner 保留，继续作为视觉锚点
- [x] 将“快速入口”前移到 Banner 下方
- [x] 快速入口仅保留高频、明确的核心能力
- [x] “今日校园”保留，但内容缩短，不再堆叠冗长信息
- [x] “和我有关”与“热门活动”重组为统一的“动态推荐”区
- [x] 首页仅保留 `1~2` 条动态推荐卡片
- [x] 热标签改为横向滑动，避免纵向继续拉长页面
- [x] 首页底部只保留低干扰的“继续探索”入口

### 4.2 模块裁剪与下沉

- [x] 下沉首页中的长摘要型模块
- [x] 下沉首页中的完整活动列表
- [x] 下沉首页中的完整热议列表
- [ ] 去掉重复功能入口，避免“快速入口”和“校园入口”双系统并存
- [x] 被下沉内容必须保留二级页跳转，不允许功能丢失

### 4.3 推荐结构收敛

- [x] 推荐区从“多张连续大卡片”改为“小规模精选”
- [x] 推荐内容优先展示与用户有关、或最值得点开的信息
- [x] 超出首页承载上限的内容统一收进“查看更多”
- [x] 首页信息流以“预览”而不是“详情列表”为原则

## 5. 视觉系统改造清单

### 5.1 卡片系统统一

- [x] 全局卡片容器收敛为 `2~3` 种
- [x] 明确“全宽主卡片”和“轻量入口卡片”的边界
- [x] 统一卡片圆角，建议控制在 `16px ~ 24px`
- [x] 统一卡片阴影强度，避免厚重投影
- [x] 减少边框、描边、底色风格混杂的问题
- [x] 避免每个模块都使用不同背景、不同容器样式

### 5.2 字阶层级优化

- [x] 一级模块标题更大、更稳、更醒目
- [x] 卡片内标题比正文有明显字重差
- [x] 正文颜色弱于标题，避免整屏都“很重”
- [x] 时间、地点、描述等辅助文本统一降级处理
- [x] 长文本严格控制行高，避免拥挤
- [ ] 多行文案限制最大行数，避免破坏节奏

### 5.3 留白系统重构

- [x] 模块与模块之间的垂直间距拉开到 `24px ~ 32px`
- [x] 卡片内部 padding 整体增大
- [x] 文本与边缘保持足够安全距离
- [x] 首页上下内容之间形成明显呼吸节奏
- [x] 避免卡片彼此贴得过近造成“窒息感”

## 6. 入口与图标改造清单

### 6.1 快速入口

- [x] 快速入口改为更轻、更整齐的网格或横滑结构
- [x] 优先展示四个核心入口：
- [x] 社团广场
- [x] 马校一站通
- [x] 帮帮我
- [x] 出物
- [x] 入口文案长度统一，避免视觉参差

### 6.2 图标系统

- [x] 快捷入口图标统一成同一套风格
- [x] 优先使用线性、克制、系统化图标
- [x] 避免花花绿绿的小图标破坏整体纯净度
- [x] 图标主色统一，不再让每个入口各自用色

### 6.3 发布入口

- [x] 首页发布入口保留，但不再制造遮挡感
- [x] 发布按钮不能压住底部 Tab 栏
- [x] 首页中的发布引导要足够清楚，但不应喧宾夺主

## 7. 体验质感改造清单

### 7.1 浏览路径

- [x] 用户进入广场后，能在 3 秒内理解页面结构
- [x] 页面阅读顺序应稳定为：看顶部视觉锚点 → 找快捷入口 → 看核心推荐 → 再决定是否继续探索
- [x] 减少用户在首页不断向下滚动才能“找到重点”的情况

### 7.2 动效与反馈

- [x] 模块进入动效保持轻量，不要过度弹跳
- [x] 横向滑动区的滚动反馈要顺滑、明确
- [x] 卡片 hover / press 态保持克制，突出精致感
- [x] 页面滚动过程应体现节奏，而不是视觉噪音

### 7.3 高级感控制

- [x] 页面主色数量控制，避免杂乱
- [x] 首页每个模块都要服从统一气质，而不是各自“抢戏”
- [x] 首屏至少留出可以让眼睛休息的空白区域
- [x] 删除任何“看起来有内容但不提升决策效率”的装饰

## 8. 推荐目标结构

建议首页最终结构收敛为：

1. Banner
2. 快速入口
3. 今日校园
4. 动态推荐
5. 热标签横滑
6. 继续探索

## 9. 验收清单

- [x] 首页纵向长度明显缩短
- [x] 首屏信息量降低，但功能路径更清晰
- [x] 页面主次关系清楚，不再出现多个模块同时抢主视觉
- [x] 卡片风格统一，不再有明显“各玩各的”问题
- [x] 字体大小、粗细、颜色形成明确层级
- [x] 页面留白显著增加，阅读不再压迫
- [x] 快速入口与底部导航的关系更协调
- [x] 发布入口不遮挡 Tab，不制造操作冲突
- [x] 用户可通过首页顺利进入主要二级模块

## 10. 当前遗留项

本轮仍建议保留以下后续优化项，不在当前任务内继续扩张：

1. “快速入口”和“继续探索”中的部分导流仍有轻微语义重叠，后续可以再做一轮入口去重。
2. 个别多行标题和摘要还没有统一做严格的行数截断，这属于下一轮精修范畴。
3. 如果后续广场首页继续桌面化增强，可以再单独评估桌面端 hover、行数控制与极端长文案场景。
