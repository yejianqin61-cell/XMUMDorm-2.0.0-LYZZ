# M01-Task003-广场统一卡片语言与容器规范

日期：2026-06-22  
优先级：P1  
状态：待开发

## 1. 任务目标

统一广场页卡片语言，减少“每个模块各玩各的”的问题，建立更克制、更高级的容器规范。

## 2. 改造范围

- 广场页全部首页卡片容器
- 快捷入口卡片
- 推荐卡片
- Hero 内嵌摘要卡片
- 可能涉及 `AppCard / ActionCard / InfoCard / MediaCard / MetricCard` 的首页样式约束

## 3. 改造要求

- 首页容器只允许保留 `2~3` 种卡片样式
- 统一圆角系统，避免不同模块圆角割裂
- 统一边框/描边/背景透明度
- 统一阴影强度，使用极轻柔投影
- 禁止单独为某个首页模块使用明显跳脱的容器风格

## 4. 验收标准

- 首页卡片视觉风格显著收敛
- 不再出现边框粗细、底色深浅、投影强弱混乱的问题
- 首页整体观感从“堆料”转为“系统化”

## 5. 测试要求

- 执行 `npm.cmd run build:web`
- 人工对比广场首页各模块卡片风格是否统一
