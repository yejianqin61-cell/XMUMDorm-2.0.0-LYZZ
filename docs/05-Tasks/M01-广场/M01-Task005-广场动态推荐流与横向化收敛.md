# M01-Task005-广场动态推荐流与横向化收敛

日期：2026-06-22  
优先级：P1  
状态：待开发

## 1. 任务目标

围绕“首页不要平铺详情列表”的原则，把广场首页剩余的信息展示改造成更轻量的推荐流和横向模块。

## 2. 改造范围

- 动态推荐区
- 热标签区
- 活动/热议的首页缩略形式

## 3. 改造要求

- 动态推荐区严格控制在 `1~2` 张主卡片
- 热标签采用横向滑动，压缩纵向空间
- 活动和热议首页若保留，仅允许用缩略形式出现
- “查看更多 / 查看全部”样式统一且低调

## 4. 验收标准

- 首页核心信息可快速扫描
- 横向模块不会打断纵向主浏览节奏
- 不再出现连续 3~4 张大卡片把页面拉得过长的问题

## 5. 测试要求

- 执行 `npm.cmd run build:web`
- 人工检查横向滚动是否顺畅、是否遮挡内容
