# M01-Task002-广场首页首屏减法与模块重排

日期：2026-06-22  
优先级：P0  
状态：待开发

## 1. 任务目标

针对当前广场首页信息密度过高、首屏模块过多的问题，先完成一次结构级“做减法”，让首页从“信息全家桶”收敛成更清晰的浏览路径。

目标是让用户进入广场页后，首屏只感知到：

1. 顶部视觉锚点
2. 高频快捷入口
3. 一个核心内容区
4. 少量可继续探索的延伸入口

## 2. 改造范围

- `frontend/src/pages/SquareHome.jsx`
- `frontend/src/pages/SquareHome.css`
- `frontend/src/components/square/TodayCampusQuickActions.jsx`
- `frontend/src/components/square/MyCampusRecommendations.jsx`
- 如有必要，补充或裁剪广场首页引用的其他展示组件

## 3. 改造要求

### 3.1 首屏减法

- 首屏模块数量必须明显减少
- 首页不再平铺所有模块详情列表
- 首页每个模块默认只展示 `1~2` 条核心内容

### 3.2 模块重排

- Banner 保留在顶部
- “快捷入口”前移到 Banner 下方
- 原底部“校园入口”四宫格与“快捷入口”重复时，优先保留更轻量的入口形式
- “和我有关”保留为核心内容区，但数量收敛

### 3.3 首页裁剪原则

- 可以先移除或下沉以下模块的首页详细展示：
  - 大段摘要列表
  - 重复入口模块
  - 首页非必要的完整活动列表
  - 首页非必要的完整热议列表
- 被裁掉的内容必须保留二级页跳转路径，不能造成功能丢失

## 4. 预期结构

建议首轮结构：

1. Banner
2. 快捷入口
3. 今日校园 Hero
4. 动态推荐
5. 热标签横滑
6. 低调的更多探索入口

## 5. 验收标准

- 广场首页纵向长度明显缩短
- 首屏滚动前能看完主要结构
- 不再出现“快捷入口 + 校园入口”双重重复模块
- 动态推荐区卡片数量不超过 2 条
- 页面结构更接近“总览页”，而不是“详情列表拼接页”

## 6. 测试要求

- 执行 `npm.cmd run build:web`
- 人工检查广场首页是否仍能进入：
  - `/about/club`
  - `/about/freshman-guide`
  - `/about/errands`
  - `/about/second-hand`
  - `/about/trending`
  - `/publish`
