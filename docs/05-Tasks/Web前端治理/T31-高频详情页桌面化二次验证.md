# T31-高频详情页桌面化二次验证

## 1. 任务目标

选择 `FoodDetail` 与 `MerchantFoodDetail` 作为高频详情页二次验证页，确认详情模板在主壳下的桌面阅读节奏已经足够稳定。

## 2. 影响范围

- `frontend/src/pages/FoodDetail.jsx`
- `frontend/src/pages/FoodDetail.css`
- `frontend/src/pages/MerchantFoodDetail.jsx`
- `frontend/src/pages/MerchantFoodDetail.css`
- 如有必要，少量与详情展示强耦合的组件样式

## 3. 不可触碰边界

- 不重写详情接口
- 不改评论、收藏、评分等核心业务流程
- 不顺手扩散到所有详情页
- 不改 `shared/*`

## 4. 实施步骤

1. 校正详情页在主壳下的正文主轴与信息区关系。
2. 对齐 `FoodDetail` 与 `MerchantFoodDetail` 的桌面节奏。
3. 降低移动端全宽单列假设。
4. 保持详情主操作与主要信息展示不回归。
5. 完成后执行 `npm run build:web`。

## 5. 验收方式

- 两个详情页在主壳下的阅读结构更稳定。
- 正文区、操作区、补充信息区层次更清晰。
- 页面不再明显保留移动端全宽详情页节奏。
- `npm run build:web` 通过。

## 6. 推荐提交信息

`refactor(web-detail): validate food detail flows for desktop`

- `T30-CanteenHome门户桌面化验证.md`

## 8. 后置影响
