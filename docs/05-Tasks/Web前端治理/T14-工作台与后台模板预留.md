# T14 工作台与后台模板预留

## 1. 任务目标

为后续 `DashboardPageLayout` 与 `AdminPageLayout` 建立预留结构与边界说明，不急于在本轮大量落地后台页面，但要保证后续有统一模板入口。

## 2. 影响范围

- `frontend/src/components/templates/DashboardPageLayout.jsx`
- `frontend/src/components/templates/AdminPageLayout.jsx`
- 或对应规划文档 / 占位实现

## 3. 不可触碰边界

- 不在本任务中批量重构后台
- 不替换 Web 主壳

## 4. 实施步骤

1. 统一工作台页和后台页的模板边界。
2. 视实际情况建立最小占位实现或结构草稿。
3. 明确后续接入页面建议顺序。

## 5. 验收方式

- 工作台与后台模板边界明确
- 后续扩展方向清晰
- `npm run build:web` 通过

## 6. 推荐提交信息

`docs(web-template): outline dashboard and admin layouts`

## 7. 前置依赖

- `T11-列表页模板落地.md`
- `T12-详情页模板落地.md`
- `T13-表单页模板落地.md`

## 8. 后置影响

- 为 Step 4 Web 主壳替换和后台治理提供接口
