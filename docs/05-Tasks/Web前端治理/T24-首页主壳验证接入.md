# T24 首页主壳验证接入

## 1. 任务目标

选择 `SquareHome` 作为主壳首个高频验证页，确认首页能落进新桌面壳里，并建立主壳与内容首页之间的稳定关系。

## 2. 影响范围

- `frontend/src/pages/SquareHome.jsx`
- `frontend/src/pages/SquareHome.css`
- 如有必要，少量首页壳接线文件

## 3. 不可触碰边界

- 不重写首页全部业务逻辑
- 不把首页一次性彻底模板化到最终形态
- 不在本任务中顺手做其它代表页

## 4. 实施步骤

1. 让首页进入新桌面壳。
2. 调整首页在主壳中的落位关系。
3. 保持首页业务能力不回归。
4. 完成后执行 `npm run build:web`。

## 5. 验收方式

- 首页在桌面端已具备网页壳承载关系
- 首页主要入口与内容节奏未回归
- `npm run build:web` 通过

## 6. 推荐提交信息

`refactor(web-shell): validate square home inside site shell`

## 7. 前置依赖

- `T23-主壳响应式退化与移动兼容.md`

## 8. 后置影响

- 为主壳正式替代旧移动式首屏提供验证基础
