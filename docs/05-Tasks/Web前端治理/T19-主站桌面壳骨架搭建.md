# T19 主站桌面壳骨架搭建

## 1. 任务目标

基于 `docs/09-Deploy/Step4-Web主壳详细设计文档.md`，先把主站桌面 Web 壳的最小骨架搭起来。

本任务不追求一步到位做完导航、右侧栏和首页改版，而是先把主壳容器层建立好，确保后续页面可以稳定挂到统一的桌面壳里。

核心目标是确认：

- 主站桌面壳有独立的 `header / sidebar / main / aside` 结构
- 壳层与页面模板层边界清晰
- 后续 `T20` 到 `T26` 有稳定的接入基础

## 2. 影响范围

- `frontend/src/components/` 下新增或收敛主壳组件
- `frontend/src/styles/` 下少量主壳样式
- 如有必要，少量 `Layout` 相关接线文件

## 3. 不可触碰边界

- 不批量迁移业务页面内部布局
- 不重写共享 API 层
- 不改动 `frontend-app/`
- 不在本任务中直接重做首页业务内容
- 不在本任务中把后台壳一起重构完

## 4. 实施步骤

1. 新增主站桌面壳的最小组件结构。
2. 明确 `header / sidebar / main / aside` 的占位与插槽关系。
3. 把壳层职责限制在容器、导航与断点，不把业务逻辑塞进来。
4. 保证桌面壳样式语言与现有 token、基础组件保持一致。
5. 完成后执行 `npm run build:web`。

## 5. 验收方式

- 主站桌面壳骨架组件已落地
- 壳层可承接页面模板层内容
- 壳层未混入页面业务逻辑
- `npm run build:web` 通过
