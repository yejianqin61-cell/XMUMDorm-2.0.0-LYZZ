# T25 工作台主壳验证接入

## 1. 任务目标

选择 `MyZone` 作为第二个代表页，验证工作台页面在新桌面壳中的承载效果。

## 2. 影响范围

- `frontend/src/pages/MyZone.jsx`
- `frontend/src/pages/MyZone.css`
- 如有必要，少量工作台相关组件

## 3. 不可触碰边界

- 不重写工作台业务逻辑
- 不顺手扩散到全部我的页面子页

## 4. 实施步骤

1. 让工作台进入新桌面壳。
2. 校正工作台模板与主壳之间的关系。
3. 保持既有业务与路由行为不变。
4. 完成后执行 `npm run build:web`。

## 5. 验收方式

- 工作台已在新主壳下稳定落位
- 工作台页面结构与主壳节奏协调
- `npm run build:web` 通过

## 6. 推荐提交信息

`refactor(web-shell): validate myzone inside site shell`

## 7. 前置依赖

- `T24-首页主壳验证接入.md`

## 8. 后置影响

- 证明主壳不仅能承载首页，也能承载工作台型页面
