# T10 筛选条模板统一

## 1. 任务目标

统一 `FilterBar` 模板，承接列表页中的搜索、标签筛选、排序和右侧工具区，避免每个列表页各写一套筛选头。

## 2. 影响范围

- `frontend/src/components/templates/FilterBar.jsx`
- 相关样式文件

## 3. 不可触碰边界

- 不批量改业务筛选逻辑
- 不把具体模块筛选项写死在模板内部

## 4. 实施步骤

1. 设计 `FilterBar` 的横向布局与换行策略。
2. 设计输入项、标签项、排序项、右侧操作区插槽。
3. 接入 token、`Input`、`Tag`、`Button` 等基础件。
4. 选取一个高频列表页做接入验证。

## 5. 验收方式

- 桌面下筛选条结构统一
- 小屏下可自然换行
- 可复用，不绑定具体业务模块
- `npm run build:web` 通过

## 6. 推荐提交信息

`feat(web-template): add shared filter bar template`

## 7. 前置依赖

- `T09-页面头部与区块头模板统一.md`
- `T05-表单基础件统一.md`
- `T06-Tag与Badge统一.md`

## 8. 后置影响

- 为列表页模板层提供统一筛选结构
