# T09 页面头部与区块头模板统一

## 1. 任务目标

统一 `PageHeader` 与 `SectionHeader` 两个高频页面结构模板，解决页面标题区、描述区、主操作区和区块标题区长期散落拼装的问题。

## 2. 影响范围

- `frontend/src/components/templates/PageHeader.jsx`
- `frontend/src/components/templates/SectionHeader.jsx`
- 相关样式文件

## 3. 不可触碰边界

- 不重写 Web 主壳
- 不批量迁移所有页面
- 不把具体接口请求写进模板层

## 4. 实施步骤

1. 建立 `templates/` 目录。
2. 设计 `PageHeader`：
   - `title`
   - `description`
   - `eyebrow`
   - `backTo`
   - `actions`
   - `meta`
3. 设计 `SectionHeader`：
   - `title`
   - `description`
   - `action`
   - `aside`
   - `compact`
4. 接入 token 与 Step 2 基础件。
5. 选取一个代表性页面头部和一个代表性区块头做接入验证。

## 5. 验收方式

- 页面标题区结构统一
- 页面内区块头结构统一
- 模板层不耦合业务接口
- `npm run build:web` 通过

## 6. 推荐提交信息

`feat(web-template): add shared page and section headers`

## 7. 前置依赖

- `T01-Token基础盘点与命名落地.md`
- `T03-Card基础件统一.md`
- `T04-Button基础件统一.md`

## 8. 后置影响

- 为 `FilterBar`、`ListPageLayout`、`DetailPageLayout`、`FormPageLayout` 提供统一上层结构
