# T23 主壳响应式退化与移动兼容

## 1. 任务目标

为新的桌面主壳补齐响应式退化和移动兼容策略，确保桌面优先方案不会直接压垮平板和手机 Web。

## 2. 影响范围

- 主壳断点样式
- 移动兼容接线
- 壳层交互开关

## 3. 不可触碰边界

- 不把整个主壳重新做成移动优先
- 不彻底删掉旧兼容入口
- 不混入大批业务页面改造

## 4. 实施步骤

1. 补齐桌面、平板、手机的退化策略。
2. 保证移动 Web 仍有可用主导航兜底。
3. 检查右侧栏、左侧栏在窄屏下的折叠行为。
4. 完成后执行 `npm run build:web`。

## 5. 验收方式

- 主壳在三类断点下都有清晰退化
- 移动兼容保留
- `npm run build:web` 通过

## 6. 推荐提交信息

`refactor(web-shell): add responsive fallback for site shell`

## 7. 前置依赖

- `T22-右侧辅助栏与主内容容器落地.md`

## 8. 后置影响

- 为代表页接入提供稳定的跨断点外壳
