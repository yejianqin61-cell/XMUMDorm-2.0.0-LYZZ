# T27 后台壳收敛与接入预留

## 1. 任务目标

在主站主壳路径稳定后，对后台壳做最小收敛与接入预留，明确它与 Step 4 Web 主壳的关系。

## 2. 影响范围

- `components/Admin/AdminLayout.jsx`
- 后台壳相关样式与接线文件
- 如有必要，少量后台路由入口

## 3. 不可触碰边界

- 不重做整套后台页面
- 不一次性统一全部后台视觉细节
- 不影响后台既有核心操作流程

## 4. 实施步骤

1. 校正后台壳与 Step 4 壳层设计语言的关系。
2. 补齐后台壳的接入预留或最小结构收敛。
3. 保持后台独立入口稳定。
4. 完成后执行 `npm run build:web`。

## 5. 验收方式

- 后台壳与 Step 4 的关系更清晰
- 后台具备继续收敛到统一壳语言的入口
- `npm run build:web` 通过

## 6. 推荐提交信息

`refactor(web-shell): align admin shell with site shell language`

## 7. 前置依赖

- `T26-列表页主壳验证接入.md`

## 8. 后置影响

- 为 Step 5 以后后台页继续治理提供结构基础
