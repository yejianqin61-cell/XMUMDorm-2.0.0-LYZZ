# T26 列表页主壳验证接入

## 1. 任务目标

选择已完成模板化的列表页作为第三个代表页，验证列表页模板在新主壳中的落位是否稳定，建议优先使用 `MerchantList`。

## 2. 影响范围

- `frontend/src/pages/MerchantList.jsx`
- `frontend/src/pages/MerchantList.css`
- 如有必要，少量列表页壳接线文件

## 3. 不可触碰边界

- 不重写列表页业务筛选逻辑
- 不顺手扩散到所有列表页

## 4. 实施步骤

1. 让列表页进入新主壳。
2. 对齐列表页模板与主壳内容区的关系。
3. 保持业务行为不变。
4. 完成后执行 `npm run build:web`。

## 5. 验收方式

- 至少 1 个列表页在新主壳下完成验证
- 页面模板与主壳容器关系稳定
- `npm run build:web` 通过

## 6. 推荐提交信息

`refactor(web-shell): validate merchant list inside site shell`

## 7. 前置依赖

- `T25-工作台主壳验证接入.md`

## 8. 后置影响
