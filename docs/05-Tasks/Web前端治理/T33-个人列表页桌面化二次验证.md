# T33-个人列表页桌面化二次验证

## 1. 任务目标

选择 `MyPosts` 与 `MyReviews` 作为个人列表页二次验证页，确认“我的”链路中的列表页已经形成统一桌面节奏。

## 2. 影响范围

- `frontend/src/pages/MyPosts.jsx`
- `frontend/src/pages/MyPosts.css`
- `frontend/src/pages/MyReviews.jsx`
- `frontend/src/pages/MyReviews.css`

## 3. 不可触碰边界

- 不重写个人列表 API
- 不修改与页面无关的广场或食堂页
- 不改 `shared/*`

## 4. 实施步骤

1. 对齐 `MyPosts` 与 `MyReviews` 在桌面壳下的页头、筛选、列表节奏。
2. 统一与 `MyZone`、`Mailbox` 的个人链路视觉落点。
3. 收口移动端页面级留白、滚动与列表容器假设。
4. 完成后执行 `npm run build:web`。

## 5. 验收方式

- 两个个人列表页在桌面主壳下结构一致性更高。
- 页面主列表、状态区、空态区节奏更统一。
- 与“我的”链路其它页面风格更连续。
- `npm run build:web` 通过。

## 6. 推荐提交信息

`refactor(web-page): align my post lists for desktop shell`

## 8. 后置影响
