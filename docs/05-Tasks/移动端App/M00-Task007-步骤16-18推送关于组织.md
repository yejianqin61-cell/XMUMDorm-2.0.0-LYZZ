## 二、推送系统 — 步骤 16

### 2.1 现状分析

| 层级 | Web 端 | RN 移动端 |
|------|--------|----------|
| 推送方式 | W3C Push API + VAPID + Service Worker | `expo-notifications` 本地定时通知 |
| 订阅存储 | `push_subscriptions` 表 | 无远程订阅 |
| 课前提醒 | 服务端定时任务每分钟轮询 | 客户端排本地通知 |
| FCM | 未实现（仅文档提及） | 未实现 |

### 2.2 迁移方案：双通道改造

#### A. 后端新增 FCM 发送通道

`services/pushSend.js` 改造为双通道：

```
sendPushToUser(userId, payload)
  ├── 查询 push_subscriptions WHERE user_id = ?
  ├── 按 channel 字段分流:
  │   ├── channel='web'  → web-push (VAPID) — 现有逻辑
  │   └── channel='fcm'  → Firebase Admin SDK — 新增
  └── 返回 { ok, sent, failed }
```

| 文件 | 操作 | 说明 |
|------|------|------|
| `services/fcmSend.js` | 新建 | FCM 发送封装（基于 `firebase-admin`） |
| `services/pushSend.js` | 编辑 | 增加 FCM 通道分流 |
| `routes/push.js` | 编辑 | subscribe 端点增加 `channel` 字段 |
| `migrations/xxx_push_channels.sql` | 新建 | push_subscriptions 表增加 `channel` 列 |
| `server.js` | 编辑 | 初始化 Firebase Admin SDK |

#### B. RN 端 FCM 注册

| 功能 | 实现 |
|------|------|
| FCM token 获取 | `Notifications.getDevicePushTokenAsync()`（Expo SDK 56 内置） |
| 向服务端注册 | `POST /api/push/subscribe { channel: 'fcm', endpoint: token }` |
| 取消注册 | 登出时 `POST /api/push/unsubscribe` |

| 文件 | 操作 | 说明 |
|------|------|------|
| `src/services/scheduleReminder.ts` | 编辑 | 增加 FCM token 注册/注销 |
| `src/services/pushService.ts` | 新建 | 统一推送服务（本地 + 远程 FCM） |

#### C. 课前提醒：本地 + 远程双保障

```
课前提醒 = 服务端 FCM 推送（主） + 本地通知（兜底）
```

- 用户已登录 + FCM token 有效 → 服务端推送
- 本地通知作为保底
