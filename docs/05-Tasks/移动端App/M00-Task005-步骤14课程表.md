## 一、模块概述

课程表模块是一个**单页面**日程查看工具，核心流程：

```
粘贴学校 AC 系统课表文本 → 后端解析 → 预览确认 → 导入覆盖 → 按天查看
```

### Web 端现状

| 维度 | 详情 |
|------|------|
| 页面数 | 1（Schedule.jsx, 655 行） |
| API 端点 | 3（week GET / import preview POST / import commit POST） |
| 数据库表 | 3（timetable_courses, timetable_meetings, timetable_import_logs） |
| 特殊功能 | 今日高亮 / 课间空档检测 / Web Push 课前提醒 / localStorage 缓存 |

## 三、API 端点覆盖（3 个）

```
GET  /api/schedule/week?week=1       — 获取周课表（按天分组）
POST /api/schedule/import/preview    — 预览解析（不保存）
POST /api/schedule/import/commit     — 提交导入（覆盖旧数据）
```

### 响应格式

**GET /week**:
```json
{
  "status": 0,
  "data": {
    "week": 1,
    "days": {
      "1": [{ "course_code": "G0173", "course_name": "新媒体", "start_time": "14:00:00", "end_time": "17:00:00", "venue": "A5#G11", "lecturer": "Dr. Li", "credit": 4 }],
      "2": [], "3": [], "4": [], "5": [], "6": [], "7": []
    }
  }
}
```

**POST /import/preview**:
```json
{
  "status": 0,
  "data": {
    "courses": [{ "no": "1", "course_code": "G0173", "course_name": "新媒体" }],
    "meetings": [{ "course_code": "G0173", "day_of_week": 1, "start_time": "14:00:00", "end_time": "17:00:00", "venue": "A5#G11" }],
    "errors": [],
    "stats": { "courseCount": 3, "meetingCount": 8, "errorCount": 0 }
  }
}
```

## 五、移动端特殊处理

### 5.1 今日检测
- 使用 JS `getDay()` 映射：周日=7，周一=1，以此类推
- 今日卡片置顶，绿色渐变背景

### 5.2 课间空档检测
- 同一天的相邻课程之间，若间隔 >= 25 分钟，插入「空档」提示
- 显示为咖啡 ☕ / 书本 📖 图标 + 时间区间

### 5.3 时间格式化
- 后端返回 `HH:mm:ss` → 前端显示 `HH:mm`（去掉秒）
- 12 小时制选项（可选）

### 5.4 系统推送 — 课前提醒（纳入本次迁移）

采用 **Expo 本地通知** 方案，无需服务端参与：

**技术栈：**
- `expo-notifications` — 本地定时通知（已内置在 Expo SDK 56）
- `expo-task-manager` — 后台任务定期重排通知

**通知触发规则（与 Web 端一致）：**
- 每节课开始前 **30 分钟** 触发推送
- 同一天同一节课只推送一次（去重）
- 仅当用户开启「课前提醒」开关时生效

**实现流程：**

```
用户导入课表
  → 解析所有 meeting (day_of_week + start_time + venue)
  → 对每个 meeting 计算提醒时间 = start_time - 30min
  → 调用 expo-notifications scheduleNotificationAsync()
  → 存入 AsyncStorage 记录已排通知列表

每周日晚上 / App 启动时
  → 后台任务检查并重排下周通知
  → 清理已过期的通知 ID
```

**权限申请：**
- 首次开启提醒时，调用 `Notifications.requestPermissionsAsync()`
- 显示系统原生权限弹窗
- 用户拒绝时，引导到系统设置

**去重机制（与 Web `class_reminder_sent` 表一致）：**
- AsyncStorage 存储已排通知 key: `reminder_{meeting_id}_{date}`
- 排通知前检查是否已存在
- Web 端已有的 `class_reminder_sent` 数据库表可同步写入

**UI 入口：**
- ScheduleScreen 顶部显示开关：「🔔 课前提醒」
- 导入成功后自动弹出「是否开启课前提醒？」

### 5.5 localStorage 缓存 → AsyncStorage
- Web 端用 localStorage 缓存课表数据减少 API 调用
- RN 端用 AsyncStorage 缓存（相同模式）
- 缓存 key: `dorm_schedule_cache_v1_w{week}`
