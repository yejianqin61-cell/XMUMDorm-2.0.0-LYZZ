# M03-Task002 - 食堂首页信息流收敛

**What to build:** 学生进入 `/eat` 后，首屏可直接搜索或选择食堂分区；继续浏览时只看到菜品推荐和食堂讨论两段连续内容，不再面对说明 hero、统计、默认轮播和内部快捷侧栏。

**Blocked by:** None - can start immediately.

**Status:** ready-for-agent

- [ ] 首页不再渲染说明 hero、统计和食堂页内部侧栏。
- [ ] 搜索与食堂分区在首屏可见，分区可进入已有商家列表。
- [ ] 无真实运营内容时不渲染轮播占位；有内容时保留现有跳转能力。
- [ ] “大家都在吃”展示已有菜品数据；随机推荐作为同区操作，不单独占用区块。
- [ ] 食堂讨论保留帖子进入与发布路径，移除解释性提示文案。
- [ ] `frontend` 生产构建通过，并完成桌面和手机宽度检查。
