# M03-Task005 - 菜品详情与点评流治理

**What to build:** 学生在菜品详情先获得图片、菜名、价格、商家和评分等决策信息，再连续阅读点评；写点评或回复不会遮挡菜品图片和正文。

**Blocked by:** M03-Task004 - 商家菜单与菜品卡统一.

**Status:** ready-for-agent

- [ ] 菜品详情移除重复快照和操作卡，收敛为单一决策信息区。
- [ ] 菜品图片占据主要视觉位置，商家信息可进入已有商家页。
- [ ] 点评改为连续阅读流，回复保持轻缩进且权限逻辑不变。
- [ ] 点评输入栏仅在点评区域内出现或吸顶，不覆盖菜品图片。
- [ ] 收藏、点赞、发布点评、删除和登录拦截行为保持不变。
- [ ] `frontend` 生产构建通过，并完成桌面、手机的详情与点评回归检查。
