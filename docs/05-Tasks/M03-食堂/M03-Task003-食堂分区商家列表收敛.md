# M03-Task003 - 食堂分区商家列表收敛

**What to build:** 学生选择食堂分区后，能以连续、可比较的商家列表找到店铺，并能按已有排序能力查看分区热门菜品，而不会被页面说明、重复统计或厚重卡片干扰。

**Blocked by:** M03-Task002 - 食堂首页信息流收敛.

**Status:** ready-for-agent

- [ ] 分区页不新增内部侧栏，只使用全站右侧辅助区。
- [ ] 商家列表统一为轻量连续行，优先展示已有可靠字段。
- [ ] 分区榜单保留进入完整榜单的路径，不重复呈现同一统计信息。
- [ ] 无商家、榜单为空或请求失败时给出一个直接返回或继续浏览动作。
- [ ] `frontend` 生产构建通过，并验证首页到分区和商家页的路由链路。
