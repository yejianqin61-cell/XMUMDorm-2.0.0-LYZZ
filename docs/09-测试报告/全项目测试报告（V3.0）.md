## 三、测试方法论

### 3.1 Mock 策略

所有测试使用 **mock 数据库** + **mock 中间件** 模式：

```
┌─────────────────────────────────────────────┐
│                  测试用例                      │
│  jest.mock('../../database') → query()       │
│  jest.mock('../../middleware/auth')          │
│  jest.mock('../../middleware/checkSanction') │
│      …                                      │
│                                              │
│  const app = createApp(router, mountPath)    │
│  const res = await supertest(app).get(...)   │
│  expect(res.status).toBe(200)               │
│  expect(res.body.data.id).toBe(42)          │
└─────────────────────────────────────────────┘
```

### 3.2 测试辅助工具（`__tests__/helpers/`）

| 文件 | 用途 |
|------|------|
| `testUtils.js` | `createApp()` / `count()` / `rows()` / `row()` / `inserted()` / `affected()` / `empty()` |
| `mockDb.js` | 原始 mock 工具（被 admin 系测试使用） |

### 3.3 运行方式

```bash
# 运行所有测试
npm test

# 运行覆盖率
npm run test:coverage

# 运行单个模块
npx jest __tests__/routes/admin.test.js --no-coverage

# 监听模式
npm run test:watch
```

### 4.2 技术局限

**跨文件 Mock 冲突：** 多个测试文件同时 mock `../../database` 时，Jest 的 `jest.mock` hoisting 机制导致 mock 互相覆盖。解决方案：

```js
// 每个测试文件添加 resetModules
beforeEach(() => { jest.resetModules(); });

// 在 beforeEach 中动态 require
const { query } = require('../../database');
const routes = require('../../routes/module');
```

受限于时间未全部应用此模式到所有模块，后续可逐模块补全。

### 4.3 前端测试

当前无前端测试。建议引入 React Testing Library + Vitest 用于组件测试，Playwright 用于 E2E 测试。
