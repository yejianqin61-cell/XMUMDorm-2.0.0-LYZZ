## 二、排行榜系统行动计划 — 自检结果

### 2.1 公式与边界

| 检查项 | 结果 | 说明 |
|--------|------|------|
| S_dish 公式 | ✅ | 与附图一致，加权平均 |
| S_shop 公式 | ✅ | 按菜品点评数加权，避免单纯累计偏大店 |
| **总点评数为 0** | ⚠️ 需约定 | 分母为 0：建议该商品 **不参与**「按综合评分」的榜单，或 S_dish 存为 NULL；实现时避免除零 |
| **店铺下无商品或全部商品 0 点评** | ⚠️ 需约定 | S_shop 分母为 0：建议该店铺 **不参与**最夯商家榜，或 S_shop 为 NULL |

**建议**：在行动计划中补充「零点评商品 / 零点评店铺不参与按评分排序的榜单」。

### 2.2 「最夯单品榜」口径 ✅ 已明确

- **已采用**：最夯单品榜 = 该商品**上线至今的累计综合评分**（全量 S_dish）从高到低；同分按上架时间越早越前；榜单每周一 0 点（东八区）刷新。
- 无需维护「本周」商品评分或本周各等级数，实现简单。

### 2.3 爆款新品「上架 &lt; 7 天」口径 ✅ 已明确

- **新品定义**：created_at 距今 **≤ 7 个自然日**的商品，**按东八区时间**计算「距今」。
- 每周一刷新榜单时按此条件筛选后按累计 S_dish 取 Top 3；「每周一重置」指榜单结果重算。

### 2.4 每周一 0 点（时区）✅ 已约定

- 所有「每周一 0 点」统一以**东八区（Asia/Shanghai）**为准，用于当周数重置、榜单刷新及爆款新品 7 天计算。

### 2.5 点评后更新顺序与范围 ✅

- 第四节已列 1～8，逻辑删除时 1～8 均需回滚，一致。
- 第六节 6.1 只写了「商品维度、店铺维度」的 1～6，**建议在 6.1 中补一句**：同时更新**店铺当周点评数**（+1/-1）与**用户当周点评数**（+1/-1），与第四节对齐。

### 2.6 同分与名额不足 ✅

- 同分按 created_at 越早越前、名额不足可为空，已写入文档，逻辑清晰。

### 2.7 实用性小结

- 当周数 + 每周一重置、S_shop 不重置只刷新榜单、门庭若市/达人用当周量，均符合「可落地、可解释」。
- 需在文档或实现中补足：零点评处理、最夯单品榜口径（建议方案 A）、爆款「7 天」定义、6.1 补充当周数更新。

## 三、对《排行榜系统行动计划》的补充建议（可直接写入原文）

以下内容建议合并进《排行榜系统行动计划》，避免实现时歧义：

1. **边界约定**  
   - 总点评数为 0 的商品：不参与「按综合评分」排序的榜单（最夯单品、爆款新品）；S_dish 可为 NULL 或不写入榜单。  
   - 店铺下无点评或 S_shop 分母为 0：不参与最夯商家榜；S_shop 可为 NULL。

2. **最夯单品榜**：评价标准 = 该商品**上线至今的累计综合评分**（全量 S_dish）；榜单每周一 0 点（东八区）刷新。  

3. **爆款新品「7 天」**：新品 = **created_at 距今 ≤ 7 个自然日**（按东八区时间）；每周一刷新榜单时按此筛选后按 S_dish 取 Top 3。  

4. **6.1 点评后更新**：已含**店铺当周点评数**、**用户当周点评数**的更新。
