# 产品需求文档（PRD）— XMUMDorm V3.0

> **提炼自：** 全项目模块开发评估报告（V3.0）、帖子系统需求与设计说明（2.0.0）

---

## 一、产品定位

XMUMDorm（厦马小筑）是面向 **厦门大学马来西亚分校（XMUM）** 学生的校园社交与生活平台。核心价值：

> 「校园里正在发生什么」— 信息聚合 + 社交互动 + 生活服务

## 二、目标用户

| 角色 | 需求 |
|------|------|
| 学生 | 浏览树洞帖子、查看食堂菜品/点评、参与社团、发布二手/跑腿、查看课表/成绩 |
| 商家 | 管理店铺/商品、查看点评、回复用户 |
| 管理员（官方号） | 管理用户、审核举报、发布公告、配置系统 |

## 三、功能模块全景

### 已完成（17 个模块，平均完成度 90%）

| # | 模块 | 核心功能 |
|---|------|----------|
| 1 | 认证系统 | 注册（学生/商家）、登录、JWT、游客模式、路由守卫 |
| 2 | 树洞/帖子 | 发帖（文字+图片）、评论（二级）、点赞、Tag 分类筛选、公告 |
| 3 | 食堂 | 区域→店铺→分类→商品四级结构、点评（含图片）、收藏、排行榜 |
| 4 | 广场 | 热搜榜、校园此刻发帖流、轮播图、四宫格入口 |
| 5 | 社团 | 创建社团、加入/关注、社团页面、动态发布 |
| 6 | 二手市场 | 商品发布、搜索、筛选、收藏 |
| 7 | 跑腿 | 任务发布、接单 |
| 8 | 一站通 | 课程评价、成绩查询 |
| 9 | 课程评价 | 课程评分、评论 |
| 10 | 课表 | 课程表展示、周视图 |
| 11 | 日记 | 个人日记 CRUD |
| 12 | 待办 | 今日待办事项管理 |
| 13 | 通知/推送 | 点赞/评论/关注通知，系统公告推送 |
| 14 | 等级系统 | 经验值累积、等级徽章、进度条、升级弹窗 |
| 15 | 组织系统 | 组织架构展示 |
| 16 | 管理员后台 | Dashboard、用户管理、举报审核、内容管理、公告、系统配置、审计日志 |
| 17 | 举报系统 | 举报提交、分类、状态跟踪 |

### 移动端（React Native）

Web 版全部功能 1:1 复刻到 React Native，后端 API 95%+ 复用。

## 四、非功能需求

| 需求 | 指标 |
|------|------|
| 性能 | API 响应 < 500ms，页面加载 < 2s |
| 可用性 | 99.5% uptime |
| 安全 | JWT 鉴权、SQL 参数化查询、文件类型/大小校验 |
| 数据 | 逻辑删除、定期备份、审计日志 |
| 国际化 | 中英双语（Tag、UI 文案） |
