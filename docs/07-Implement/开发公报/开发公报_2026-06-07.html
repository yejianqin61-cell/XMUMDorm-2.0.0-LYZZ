四、今日开发难题 & 解决方案

🔴 难题 #1：Android Safe Area — 状态栏遮挡内容（耗时最长）

现象：APK 安装到真机后，所有页面顶部文字和状态栏（时间、电量、信号）重叠，登录页标题被遮挡。
尝试过的方案（均失败）：

❌ CSS padding-top: 28px 硬编码不同机型状态栏高度不同（24-48px），硬编码不适配
❌ CSS env(safe-area-inset-top)Android WebView 永远返回 0，此 CSS 函数在 Android 上无效
❌ StatusBar.setOverlaysWebView({ overlay: false })Android 15+ Edge-to-Edge 强制 overlay，此 API 被无视
❌ @capacitor/status-bar 动态 importVite rollupOptions.external 导致 WebView 中模块无法解析
❌ styles.xml 加 windowTranslucentStatus与 Capacitor WebView 主题冲突，App 闪退
❌ window.Capacitor.Plugins.StatusBar 立即调用Capacitor 桥未初始化完成时访问，返回 undefined

✅ 最终解决方案

setTimeout(500ms) + window.Capacitor 全局 + .then().catch() 模式

// 核心逻辑（main.jsx 末尾） setTimeout(function () { try { var C = window.Capacitor; // 不用 import，直接读全局 if (!C) return; document.body.classList.add('capacitor-native'); var SB = C.Plugins && C.Plugins.StatusBar; if (SB) { SB.getInfo().then(function (info) { // .then() 非 async/await document.documentElement.style.setProperty('--safe-top', info.height + 'px'); }).catch(function () {}); } } catch (_) {} }, 500); // 等 Capacitor 桥初始化

🔴 难题 #2：Vite 构建与 Capacitor 模块解析冲突

现象：npm run build:capacitor 报错 "Could not resolve @capacitor/app"。
根因：@capacitor/* 包安装在根目录 node_modules，Vite 在 frontend/ 目录构建时找不到这些依赖。
解决：vite.config.js 添加 build.rollupOptions.external: (id) => id.startsWith('@capacitor/')。这意味着 Vite 不打包这些模块——它们在 Capacitor WebView 运行时由系统提供。同时 Web 端永远不会触发这些 import。

🟡 难题 #3：Android 15 Edge-to-Edge 强制模式

现象：targetSdkVersion=36 触发 Android 15+ 强制 Edge-to-Edge。状态栏透明 + 导航栏透明，内容从屏幕顶部 (0,0) 开始绘制。
影响：所有依赖"WebView 自动在状态栏下方"的假设失效。
解决：JS 动态读取 StatusBar.getInfo().height 注入 CSS 变量 + 100vh → 100dvh 全局替换。

五、经验教训

1. Capacitor WebView ≠ 浏览器

CSS env(safe-area-inset-*) 在 Android WebView 中无效。ES Module 动态 import() 解析行为不同。必须用 window.Capacitor 全局 + JS 桥接方案。

2. Vite external 的双刃剑

external 让构建通过，但运行时 @capacitor/* 模块不存在于浏览器的模块解析器中。结论：Vite 构建的 Capacitor App 不能使用 import('@capacitor/xxx')，只能用 window.Capacitor。

3. setTimeout 不是 hack，是必要等待

Capacitor 桥在 WebView 中异步初始化。React 渲染完成 ≠ Capacitor 插件就绪。500ms 延迟是保证桥已建立的必要条件。

4. .then().catch() 比 async/await 更安全

在 Capacitor WebView 的初始化阶段，async/await 的 Promise rejection 可能在某些情况下无法被 try/catch 捕获。.then().catch() 显式处理了两个分支。

5. 100dvh 是移动端新标准

100vh 在 Android 15+ Edge-to-Edge 下计算错误（包含状态栏和导航栏高度）。100dvh 动态排除系统 UI 高度，2025+ 移动端必须使用。

6. 构建隔离保护 Web 体验

npm run build:web（绝对路径）和 npm run build:capacitor --base=./（相对路径）彻底分离。Web 端 108 tests 零影响证明了这个策略的正确性。
